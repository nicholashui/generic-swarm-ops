想像一下，你想打造一個真正值得信賴、能夠把事情做好的 AI agent。它不只是回答一條問題咁簡單，而是要一步一步處理複雜任務，懂得運用工具、自己做決定，仲要喺過程中不斷修正自己。呢個正正就係我哋今日要探討嘅主題：點樣設計一個 agent loop，令佢可靠、令你睇得明白、甚至可以隨住時間自己進步。

我哋由大方向講起。你建立嘅每一個 agent loop，都應該符合三個核心特質。佢要受控，唔好失控咁觸發一連串無法收拾嘅連鎖反應。佢要可觀察，等你喺任何一刻都睇到內部發生緊咩事。佢亦要可演化，等佢可以成長、變得更好。要做到呢啲，你就要俾 agent 一個明確嘅狀態去追蹤進度，俾佢結構化嘅輸入同輸出，令一切保持乾淨同可預測，喺接受任何結果之前都要有強制嘅品質檢查，要有能力察覺自己係咪喺度兜圈，要有出事時會跳掣嘅安全開關，仲要有刻意安排嘅時刻去停低、整合同反思。

咁呢一切由邊度開始呢？由一個叫做 ReAct 嘅模式開始，ReAct 即係 Reason 加 Act，推理加行動。你可以將佢諗成現代 agent 嘅心跳。呢個循環非常簡潔。首先係 thought，即係思考，agent 喺度推敲目標、自己行到去邊、仲欠缺啲咩、下一步要做乜。跟住係 action，行動，佢真係做一件事：運行一個工具、將工作交俾另一個 agent，或者決定自己完成咗。再跟住係 observation，觀察，即係由外界、工具或者 sub-agent 返返嚟嘅結構化結果。呢個結果會加入歷史紀錄，然後成個循環再重複。

點解呢個做法會勝過淨係喺腦海裏面靜靜咁推理，又或者盲目咁採取行動呢？因為思考嗰部分令 agent 可以提前規劃、應付突發情況、需要時改變方向。而行動嗰部分就將推理錨定喺真實、觀察到嘅結果上面，呢樣大大減少咗 agent 自己亂作嘢嘅情況。

而家睇吓呢套嘢喺真正大規模運作時係點樣嘅。喺 xAI，呢啲 agent loop 喺伺服器端運行，支撐住好似網頁搜尋、平台內搜尋同程式碼執行呢類工具。佢哋亦都用多 agent 協作，一個請求可以啟動四個甚至十六個專門嘅 agent 即時合作，再由一個領導 agent 將佢哋所有結果整合成一個連貫嘅答案。仲有一種計劃優先嘅風格，系統會先規劃出一個方案，然後派出並行嘅 sub-agent 同時處理唔同部分。

對於真正複雜嘅任務，你就要再行多一步，進入分層、可自我演化嘅系統。喺呢度你會有一個中央嘅 orchestrator，即係統籌者或者規劃者，佢將任務拆解成幾個部分，將每一部分交俾一個運行緊自己循環嘅專門 sub-agent，然後接收返嗰啲冒返上嚟嘅結構化結果。統籌者會將一切整合，做品質檢查，甚至可以透過反思啱啱發生過嘅軌跡嚟改善自己。咁就令你可以擴展規模而唔會失去控制。

當然，如果我哋忽視咗 agent 通常會點樣失敗，呢一切都無意思。研究人員已經歸納咗主要嘅失敗類別，值得我哋認識。最大嗰個，佔咗超過四成嘅問題，就係規格同設計：指示含糊，亦都冇講清楚成功究竟係點樣。解決方法就係喺一開始就驗證嘅結構化任務規格。跟住係無限循環同空轉，agent 不斷重複同樣嘅動作但毫無進展。你可以用循環偵測、對步數設一個硬性上限，同埋進度檢查點嚟防範。仲有 context 爆炸，即係重要資訊喺一個又長又散亂嘅歷史裏面唔見咗。答案就係分層記憶、結構化狀態同摘要。仲有驗證失敗同幻覺，即係未經檢查嘅輸出令錯誤越積越多；你可以用專門嘅驗證者同評審 agent，加上結構化觀察嚟對付。仲有協調崩潰，角色互相衝突、狀態過時，呢個就需要一個強大嘅統籌者同清晰嘅資訊契約。最後就係終止問題，agent 唔係停得太早，就係永遠唔停，可以用一個明確嘅完成動作同品質關卡嚟解決。如果你淨係修好少數幾樣嘢，就修好呢幾樣：結構化規格、強制嘅驗證層，同埋對於幾時停低嘅明確控制。

咁你究竟點樣將佢哋全部砌埋一齊呢？等我哋逐個階段行一次成個流程。

由階段零開始，初始化，而且係以規格為本。你將指示變成一個結構化嘅任務規格：目標、成功準則、限制、輸出格式、預算同品質門檻。你設定 agent 嘅狀態，包括佢嘅任務筆記、待辦清單、記憶、追蹤器，同一個版本登記冊。理想嘅話，你仲會生成一個高層次嘅計劃並加以驗證。然後你決定架構：你需要一個簡單嘅扁平循環，定係一個完整嘅層級結構？

跟住係階段一，核心嘅受控循環。喺任務未完成之前，agent 會觀察自己當前嘅狀態，如果 context 變得太長就摘要一下。佢形成一個 thought，決定下一個動作，無論係用工具、委派、整合定係完成。佢執行嗰個動作，外面包住好似重試同斷路器呢類安全措施。佢收集一個結構化觀察，更新自己嘅狀態同待辦清單，亦都不時做少少反思。當成功準則達成而且通過品質關卡，當佢去到最大步數，當佢明確宣布完成，又或者當佢遇到一個無法復原嘅錯誤，呢個循環就會結束。

階段二係分層委派。統籌者為一個子任務寫一份精細嘅規格，交俾一個運行緊自己完整循環嘅 sub-agent，而嗰個 sub-agent 返返一個結構化結果，冒返上嚟做整合。

階段三係整合同品質關卡。喺呢度你將嚟自多個分支嘅結果集合埋一齊，用一個驗證者或者評審 agent 審視佢哋，將一切協調好，解決任何衝突，按需要重新組織，再更新全局嘅計劃同狀態。

階段四係反思同自我演化。Agent 分析自己執行過程嘅軌跡，診斷出咗咩問題，然後提出有針對性嘅改善，可能係佢嘅提示、佢嘅工具，或者佢嘅 agent 配置。最關鍵嘅係，佢喺將呢啲改動定為新版本之前會先驗證，而且保留有需要時回滾嘅能力。

而階段五係終止同輸出。Agent 做一次最終整合，產生符合原本規格嘅結構化輸出，再儲存完整嘅軌跡同版本歷史，既為審核，亦為將來嘅學習。

而家等我哋將其中幾個機制講得具體啲，因為設計嘅精髓就喺啲細節裏面。

就以受控循環本身為例。喺佢運行嘅時候，佢會一路記住步數，超過最大值就會停。每行一轉，佢會為最近嘅歷史取一個指紋，如果佢之前見過一模一樣嘅指紋，佢就認得自己卡咗喺一個循環度，於是強制重新規劃，而唔係永無止境咁兜圈。佢建立 context，叫模型做決定，並且預期收到一個乾淨、結構化嘅決定：一個 thought、一個動作類型同一個 payload。如果決定係完成，佢會先將輸出對照規格做驗證，真係通過先至返回；否則就繼續行落去。如果動作係用工具或者委派，佢就喺斷路器同重試邏輯後面安全咁執行。而如果有任何嘢拋出錯誤，佢就將錯誤類型、訊息同完整細節用結構化形式記低，咁失敗本身就變成有用嘅資訊，而唔係一條死路。每一步，個 thought、個動作同個觀察都會加入歷史，待辦清單亦都會更新。

斷路器值得我哋睇真啲，因為佢係個安全閥。佢會追蹤失敗，並且處於三種狀態之一。當佢閉合，一切正常流動。如果失敗累積到超過一個門檻，佢就會打開，請求停止流動，等系統唔好不斷狂撼一個已經壞咗嘅嘢。經過一段復原等待時間之後，佢會轉去半開狀態，喺度小心咁放行有限數量嘅測試呼叫。如果嗰啲成功，佢就重設返閉合，正常運作恢復；如果失敗，佢就再次打開。呢個簡單嘅模式，就係防止一個出事嘅依賴項拖垮成個 agent 嘅關鍵。

跟住係驗證者，或者評審 agent。佢嘅職責就係要嚴格。你俾佢一個候選結果同任務規格，佢就返一個清晰嘅判斷：呢個過唔過得到關、分數係幾多、有咩問題、有咩建議可以改善。呢個結構化嘅判斷，就係支撐住成個系統各處品質關卡嘅動力。

而自我演化嗰一步就為改善畫上一個閉環。Agent 睇住一條軌跡，診斷問題，對某個組件提出一個修改，然後喺將佢登記為新版本之前，先驗證呢個改動係咪真係令情況變好。

如果你諗緊究竟點樣可以建立呢一切而唔會手足無措，其實有一個合理嘅路線圖。第一個星期，集中喺結構化規格同一個帶循環偵測嘅基本 ReAct 循環。第二個星期，加入驗證者同評審，連同結構化觀察。第三個星期，引入分層委派同斷路器。由第四個星期開始，就逐層加入自我演化、完整追蹤同反覆精煉。

呢一切都建基於一個深厚嘅研究同實戰根基：最初嘅 ReAct 研究、歸納多 agent 系統點樣崩潰嘅失敗分類、喺 xAI 運行嘅生產級 agentic 系統、為呢個目的而建嘅系統嘅協作模式，以及教 agent 自我評審同改善嘅反思框架。

結論就係咁。一個出色嘅 agent loop，唔係放任一個模型亂咁跑然後祈求好運。佢講求嘅係刻意嘅設計：明確嘅狀態、結構化嘅溝通、真正嘅驗證，同埋內建嘅安全。將佢建造成受控、可觀察、可演化，你就會得到一個你真正可以信賴去做實事嘅 agent，亦係一個每次運行都會靜靜咁變得更好嘅 agent。
