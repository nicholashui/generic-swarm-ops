我哋今次傾下點樣建立一個可靠、達到生產級數的 agent loop。呢個係經過優化嘅第三版設計，佢喺之前所有版本嘅基礎上，加入咗一樣新嘅嘢：人類真正嘅思考方式。佢嘅核心理念好簡單，但實際用起上嚟好強大。與其畀語言模型喺一個失控嘅連鎖反應入面亂咁走，不如畀佢一個受控嘅迴圈。即係要有明確嘅狀態、結構化嘅輸出、喺每個關卡都有質素檢查，仲有一個清晰嘅層級架構，由規劃者將工作分派畀專家。呢個係有管理嘅協調，唔係失控嘅反應。結果會逐層向上滙集，經過整合，再變成一個經過深思熟慮嘅綜合成果。

咁呢啲理念由邊度嚟呢？佢哋建基於扎實嘅研究。有二零二三年一篇著名論文提出嘅 ReAct 範式，有研究多代理系統點解會失敗嘅分析，有關於反思同自我改進嘅著作，有記憶方面嘅研究，仲有開發者實際遇到無限迴圈同上下文膨脹嘅報告。喺呢一切之上，第三版仲加入咗對四十幾種人類認知框架嘅系統性回顧，並且按照佢哋對 agent loop 嘅實用程度排咗序。

我哋由基礎講起，就係 ReAct 範式。核心理念係將兩樣嘢交織埋一齊。第一樣係用文字表達嘅推理，我哋叫佢做 thought，即係思考。第二樣係行動，例如呼叫工具、同環境互動，或者將工作交畀另一個 agent。每次行動之後，你都會得到一個觀察結果，而呢個觀察會將推理錨定落實同更新。點解呢個組合咁有效呢？如果你淨係推理，冇外部嘅依據，模型就會傾向產生幻覺，錯誤亦會像雪球咁愈滾愈大。如果你淨係行動，冇高層次嘅規劃，你就會得到笨拙、低效嘅行為，亦處理唔好突發情況。但係將佢哋結合起嚟，神奇嘅效果就出現喇。思考會分解目標、追蹤進度、處理例外情況。行動就帶返真實嘅觀察結果，去修正推理。研究顯示，喺互動式任務上呢個做法帶嚟有意義嘅雙位數提升，同時減少幻覺。

一個完整嘅循環係咁樣嘅。模型先有一個思考，推敲一下當前嘅狀態同下一步。然後佢採取行動，輸出一個可執行嘅步驟，無論係呼叫工具、分派工作，定係發出已完成嘅訊號。跟住就係觀察，一個結構化嘅結果，附帶有用嘅元數據，例如狀態、信心度、摘要同任何問題。呢個結果會加入到正在進行嘅歷史記錄入面，然後迴圈再重複。當有嘢失敗咗，思考嗰一步就會察覺到，例如話「冇任何有用嘅嘢返嚟」，然後喺下一次循環就會重新規劃。

呢個唔淨止係理論。Grok 系列嘅模型喺伺服器端就運行緊呢個迴圈嘅生產級版本。模型自己決定用邊啲工具，喺幕後執行，例如網頁搜尋同程式碼執行，然後不斷迭代，直到佢可以產生最終答案。客戶端淨係見到完成嘅輸出，加上選擇性嘅推理符記。

而家我哋擴展到生產級嘅多代理協調。想像一個系統可以啟動可配置嘅 agent 團隊，可能用四個去做快速集中嘅工作，用十六個去做深入全面嘅研究。呢啲專門化嘅 agent 並行運作，每個都即時貢獻推理、工具呼叫同發現。然後有一個領導 agent 綜合成個討論，互相對照所有資料，再交出最終嘅結構化答案。各個子代理嘅內部狀態預設係隱藏嘅，目的係為咗控制同安全，所以淨係領導嘅輸出會展示出嚟。呢度嘅強項就係深入嘅多步驟研究、好似表格同比較咁嘅結構化輸出，仲有自動使用工具，而客戶端完全唔需要介入迴圈。相關嘅工具仲採取規劃優先嘅做法，先生成一個明確嘅計劃，然後先至分散畀喺隔離環境入面工作嘅並行子代理。

下一層就係層級架構同自我演化。最頂層坐住一個中央規劃者，又或者叫協調者、監督者。佢將目標拆解成子任務，再分派畀專門化嘅子代理：一個深度研究員、一個分析員、瀏覽器同工具 agent、一個報告員等等。每個子代理都運行自己嘅迴圈。路由跟住一個樹狀結構，結果就逐層向上滙集。呢度有個啟發，就係將工具、環境同 agent 都當成第一級、有版本管理、有生命週期管理嘅實體，全部都跟住標準化嘅協定。仲有一個用嚟自我演化嘅閉環回饋：系統會反思自己嘅執行軌跡，歸因錯誤，提出修改，喺保留嘅樣本上驗證，再登記改進咗嘅版本，而且可以回退。一個追蹤器會記錄完整嘅執行軌跡，呢樣對審計同優化都有用。最後，規劃者會整合啲子結果，協調證據，解決衝突，再更新全局計劃。用呢種方式建立嘅系統，喺艱難嘅基準測試上已經達到大約九成嘅成績，而自我演化仲帶嚟更多提升。

所以成個優化後嘅模型係咁樣嘅：由 ReAct 核心迴圈開始，喺上面為複雜嘅工作加一層層級分派，然後再加入一個明確嘅規劃階段、反思同批判嘅關卡，以及帶版本管理嘅結構化狀態，去達到生產級嘅可靠性。

而家講下第三版真正全新嘅嘢：由嗰啲排咗序嘅人類思考模型而來嘅認知架構強化。呢啲唔係事後補上去嘅附加品，而係第一級嘅機制，佢哋帶嚟四樣嘢：透過情境感知路由帶嚟嘅適應性智能、透過喺行動前思考風險帶嚟嘅前瞻性穩健、透過快慢思考路徑帶嚟嘅高效認知，以及透過結構化反思帶嚟嘅更深層學習。等我同你逐個講下最重要嗰幾個。

第一個係 Cynefin 框架。呢個排第一位。喺定好規格之後，以及每次決定要唔要重新規劃嗰陣，系統都會將任務嘅情境分類。係咪簡單，因果關係清晰？係咪繁雜，需要專家分析？係咪複雜，事情會無法預料咁浮現出嚟？定係混亂，係一個真正嘅危機？根據呢個分類，佢就會調校個迴圈。簡單同繁雜嘅任務行快速路徑，關卡較輕。複雜同混亂嘅任務就會得到全面嘅深思熟慮處理，重度反思、多個批判者，以及更深入嘅診斷。呢個單一理念，喺平衡效率同可靠性方面，係最高槓桿嘅新增功能。

第二個係事前驗屍分析。喺計劃生成之後但仲未確定之前，系統會做一個強制性嘅練習：想像呢個計劃喺六到十二個月之後徹底失敗咗，然後逆向推敲搵出最主要嘅成因，再喺規格、成功準則或者 agent 角色入面明確噉去化解佢哋。呢樣可以由協調者執行，又或者由一個專責嘅紅隊批判者執行。佢用接近零嘅成本，透過前瞻性嘅風險模擬去強化規劃。

第三個係事後檢討，喺每個里程碑同終止嗰陣使用。佢跟住一個簡單嘅四條問題範本。原本應該發生啲咩？實際上發生咗啲咩？點解？同埋下一步點做？最後嗰一步將教訓變成具體嘅演化行動，直接餵返畀自我改進。

第四個係雙環學習。喺事後檢討提出咗戰術性嘅修正之後，系統會問一條更深層嘅問題：係邊啲主導嘅假設帶我哋到呢個地步？啲提示範本、成功嘅定義、agent 角色嘅界線、驗證嘅門檻，甚至係拆解任務嘅方式。呢啲喺更高嘅層次上應唔應該改變呢？要到呢個時候，佢先至確認帶版本嘅更新。呢樣先至令到自我演化真正強大，而唔係淨係修補表面症狀。

第五個係識別啟動嘅決策，再配搭雙重歷程理論，即係快思同慢想嘅概念。呢個引入咗一條快速識別路徑。喺投入冗長嘅推理之前，系統會查詢一個模式庫，即係一個儲存咗成功軌跡嘅長期記憶。如果搵到一個強力嘅匹配，而情境又容許，佢就會做一個輕量級嘅心理模擬，類似「呢個同過往一個成功嘅軌跡好相似，咁就採取嗰個行動」，然後用極少嘅符記繼續落去。呢個就係系統一，快速而直覺，用嚟做例行嘅專家工作。系統二就係緩慢而審慎，即係完整嘅 ReAct 迴圈，用嚟應付新奇或者高風險嘅情況。元認知會決定用邊一個，並且喺信心唔足嗰陣回退到完整迴圈。

講開又講，第六個就係元認知循環，一個喺每次迭代旁邊運行嘅輕量級並行程序。佢會規劃、監察偏差同情境是否吻合、評估嚴謹程度，再即時作出調整，可以切換模式、提早重新規劃，或者升級某個關卡。

跟住係根本原因分析：五個點解、石川魚骨圖，同埋故障樹。喺持續失敗嗰陣，系統會用反覆嘅點解去深入挖掘，再喺多個維度將根本原因分類，例如人同提示、流程同方法、模型同工具、數據、環境同指標。仲有紅隊思維，一個專責嘅批判者去對抗性噉攻擊一個計劃或者輸出，從而揭示隱藏嘅弱點。仲有 Paul-Elder 批判思考框架，佢用思考元素同智性標準，例如清晰、準確、精密、相關、深度同公平，去豐富驗證器同思考嗰一步。仲有制約理論，配搭 TRIZ 嘅創新解難原則，喺整合階段用嚟解決互相衝突嘅目標。最後，好似六頂思考帽同 SCAMPER 檢查清單咁嘅選擇性多角度檢視，可以幫創意型嘅子代理減少盲點。

為咗支援呢一切，記憶架構會升級，加入一個模式庫，儲住按情境類型標籤嘅過往軌跡嘅向量同元數據。驗證器而家可以喺唔同嘅批判模式下運行，標準、紅隊、Paul-Elder 或者六頂帽，甚至可以做集成。任務規格亦多咗一個認知檔案，附有啟用快速路徑、揀選反思風格同批判模式嘅旗標。最關鍵嘅係，所有呢啲新步驟都係有界限、有版本、有記錄，並且可以按任務開關或者限制深度。佢哋將 agent 由一個能幹嘅引擎，轉化成一個會思考自己思考方式、會預料失敗、會按情況調整自己審慎程度嘅系統。

咁，呢一切點解咁緊要呢？因為研究好清楚噉指出咗 agent 系統點解會失敗。分析師研究咗超過一百五十條跨越熱門框架嘅軌跡，發現大部分失敗都源於設計同規格嘅問題，當中超過四成係咁，再加上協調崩潰同薄弱嘅驗證。通常都唔係模型本身智能嘅問題。等我哋逐個講下主要嘅問題類別同佢哋嘅解決方法。

最大嘅類別係規格同設計上嘅含糊。Agent 會違反或者誤解含糊嘅規格，角色唔清晰，亦冇成功準則，於是好早就走錯路，錯誤就不斷累積。解決方法就係喺一開始有一份強制性嘅結構化任務規格，列明成功準則、限制、輸出格式同質素門檻，並且當佢係一份可以更新嘅活文件。喺迴圈開始之前加入自動嘅規格驗證，並且喺協調者同子代理之間定下清晰嘅角色同資訊契約。

跟住係無限迴圈同打轉，即係 agent 不停重複同樣嘅行動但毫無進展。呢樣浪費符記、引致逾時，亦令人人都好沮喪。解決方法就係循環偵測，將最近嘅行動同觀察做雜湊，如果相似度超過某個門檻就強制重新規劃。設定明確嘅步驟同反思回合上限，用基於進度嘅提早退出，再要求一個必須通過驗證先會被接受嘅完成行動。喺層級架構入面，協調者會監察子代理，可以終止或者重新分派卡住咗嘅分支。

第三係上下文視窗爆炸，即係過長嘅軌跡令到早期嘅資訊被丟失，引致前後不一同目標漂移。解決方法就係積極嘅層級記憶，短期工作記憶配搭一個有向量搜尋嘅長期持久儲存。喺里程碑做摘要，保持結構化嘅狀態，淨係留低關鍵事實，而唔係每一回合都倒晒成份歷史出嚟，再淨係畀子代理相關嘅上下文片段。

第五係代理之間嘅錯位，例如越權扮演角色、目標衝突，同埋過時嘅共享狀態。解決方法就係一個有明確拆解同路由嘅強力中央協調者、清晰嘅資訊契約、帶版本嘅共享狀態，以及斷路器去偵測不一致並暫停，從而調和或者升級。

第六係終止同目標漂移，即係太早停止，又或者認唔出工作已經完成。解決方法就係明確嘅成功準則、一個必須通過驗證器嘅專責完成行動，以及定期對照原本目標做對齊檢查。

而家等我哋逐個階段行一次成個 agent loop。

第零階段係初始化，即係規格驅動嘅設置。目標係喺任何迭代開始之前，先確立一個清晰嘅契約。你將人類嘅指令解析成一份結構化嘅任務規格，包含目標、成功準則、限制、輸出格式、預算同質素門檻。你建立初始狀態：一個有當前計劃同待辦清單嘅結構化便箋、一個有短期同長期部分嘅記憶層、一個追蹤器，同埋一個版本登記冊。對於複雜嘅任務，你可以選擇生成一個高層次嘅計劃，再對照規格驗證。跟住就係第三版嘅認知強化：用嚟為任務做路由嘅 Cynefin 分類，同埋事前驗屍分析，呢個而家對除咗最簡單以外嘅所有任務都係一個建議嘅強制關卡。最後，你決定架構，係扁平、層級定混合，再設定初始嘅認知檔案旗標。

第一階段係核心迭代迴圈。只要任務未終止，你就會一直行落去。喺每次循環開始嗰陣，系統會揀一個模式。如果情境簡單、有強力嘅模式匹配，而信心又高，佢就會行快速識別路徑，做一個用極少符記嘅輕量級思考。如果唔係，佢就會進入全面嘅深思熟慮模式，做詳細推理同全套關卡，同時元認知會運行一個並行嘅監察器，如果不確定性突然飆升就可以強制切換模式。喺每次迭代入面，你觀察當前狀態，如果上下文太長就積極做摘要。你推理，同時用元認知做覆蓋層去檢查偏差同進度。你決定下一個行動，係呼叫工具、分派畀子代理、喺內部更新狀態，定係完成。你執行並收集一個結構化嘅觀察，附有狀態、數據、摘要、信心度、問題同建議。每隔幾步你就做一次輕量級反思，快速檢查一下軌跡係咪仲對齊。

呢度有個好實用嘅生產模式，就係斷路器。佢喺一個工具或者子代理不斷失敗嗰陣，防止連鎖故障。佢有三個狀態。閉合即係正常運作，請求可以通過。斷開即係已經失敗咗太多次，所以佢即刻快速失敗去保護系統。半開就係喺逾時之後，佢容許幾個測試請求去睇下係咪已經恢復。如果測試成功，佢就重新閉合；如果失敗，佢就重新斷開。你每個工具類型或者每個子代理角色用一個斷路器，再配搭用指數退避嘅重試包裝。呢個實現亦包含將最近歷史做雜湊嘅循環偵測，同埋一個支援嗰啲唔同批判模式嘅驗證器。終止條件包括達到成功準則、用盡預算、一個明確而經過驗證嘅完成行動、一個無法挽救嘅失敗，又或者喺結果已經滿足目標嗰陣提早退出。

第二階段係層級分派同子迴圈。當協調者決定要分派嗰陣，佢首先做拆解同路由，揀選合適嘅專家子代理，再建立一份淨係帶相關上下文片段嘅狹窄子任務規格。子代理跟住運行自己嘅獨立迴圈，維持自己嘅本地狀態，亦可以進一步分派或者呼叫工具。佢返回一個結構化嘅結果，逐層向上滙集，當中包含來源，即係邊個 agent 做嘅同一個軌跡摘要。父層將佢記錄低，驗證並整合，合併分支同解決衝突，更新全局計劃，再決定下一步。而喺依賴關係容許嘅地方，獨立嘅分支可以並行運行。

第三階段係整合同綜合。喺啲子結果返嚟之後或者喺重大里程碑嗰陣，你滙集所有相關嘅觀察。然後你做協調，由一個語言模型或者一個專責嘅報告員 agent 去合併、去重、互相對照同解決矛盾，整合成一個統一嘅視圖。你將佢重組成目標嘅輸出形態，並執行規格入面嘅格式。跟住就係質素關卡：一個批判者對照成功準則為成果評分，並檢查有冇幻覺同缺口。如果唔合格，你就觸發一個優化迴圈。如果合格，你就繼續落去。然後你將整合咗嘅知識持久化到長期記憶。

第四階段係反思、批判同自我演化，而呢度就係第三版真正出色嘅地方。而家每次反思都跟住一個明確嘅多層協定。佢由強制嘅事後檢討範本開始，即係嗰四條問題：原本應該發生啲咩、實際發生咗啲咩、點解，同埋下一步。對於「點解」，會有第一輪嘅標準歸因，而對於更難嘅問題，就會有深入嘅一輪，運用五個點解，再用魚骨圖將成因分類，跟住用 Paul-Elder 嘅視角去交叉檢查。之後就係雙環學習嗰層，問下係邊啲主導假設造成咗呢個結果嘅條件，淨係捱得過嗰條元問題嘅改動先會被提出。然後自我演化迴圈就啟動：收集軌跡、喺嗰個框架入面歸因錯誤、提出針對性嘅改動、喺保留嘅任務上重新執行去驗證，如果有改善而又通過雙環批准，就登記新版本，連同佢嘅血統同理據，並更新模式庫畀未來嘅快速匹配用。批判者 agent 支援所有嗰啲模式，紅隊模式喺高風險工作上特別受推薦。結果就係第四階段變成一個真正令智能複利增長嘅引擎，而唔係淨係漸進式嘅修補。一個好提示：將事後檢討嘅輸出儲存做連結住版本嘅結構化產物，建立一個可審計嘅學習歷史，畀未來嘅 agent 可以檢索。

第五階段係終止同輸出。當啲關卡通過，你就做最後一次綜合，產生符合規格嘅結構化最終輸出，可以選擇加一個反思摘要，再將完整嘅軌跡同版本持久化，以供審計同重播。而喺成個過程入面，喺質素關卡失敗、高風險行動或者預算耗盡嗰陣，都有人為介入嘅鈎子。

講幾句關於基礎設施嘅嘢。你嘅狀態架構結合咗任務規格、計劃同待辦、思考-行動-觀察三元組嘅歷史、記憶、版本同追蹤器。記憶應該係層級式同會話隔離嘅，喺上下文壓力下做摘要。追蹤捕捉完整嘅執行圖，用嚟除錯同優化。版本管理涵蓋提示、工具、agent 角色同產物，全部都有血統同回退。而生產級嘅加固，意思就係嚴格嘅輸出架構、逾時同重試、成本監察、可觀察性，以及沙盒化嘅執行。

咁幾時應該用邊樣呢？對於簡單嘅事實查詢，一個扁平嘅單一 ReAct 迴圈就夠喇。對於多步驟研究，就用多代理或者層級架構，有並行嘅 agent 同一個負責綜合嘅領導。對於編程或者長期項目，就用規劃優先，配搭喺隔離工作樹入面嘅層級子代理同一個待辦檔案。對於開放式嘅創意工作，就將 ReAct 同反思同自我演化結合埋一齊。而對於高風險、需要可靠嘅自動化，就用以上全部，再加強力嘅質素關卡。

總結一下常見嘅陷阱同佢哋嘅解決方法：用積極嘅摘要同層級狀態去處理上下文爆炸；用硬性嘅迭代上限同進度追蹤去制止無限迴圈；用結構化嘅子結果同一個專責報告員去修正差劣嘅整合；將每一個重大聲稱都落實有依據，去避免計劃入面嘅幻覺；用明確嘅子任務規格去防止脆弱嘅分派；以及確保有完整嘅追蹤去提供可見度。

咁實際上點樣開始呢？由你揀嘅框架入面一個最簡嘅 ReAct 框架開始。加入一個結構化嘅觀察架構同待辦同狀態管理。再加一層層級分派同整合。裝置追蹤同質素關卡。喺重複嘅任務類型上試驗反思同自我演化。再為研究密集嘅子任務整合一個多代理模式。成個設計都係要做可執行嘅指引。複製啲模式、調整啲虛擬碼，再不斷迭代。最大嘅領悟就係：可靠性唔係淨靠原始嘅模型算力而來嘅。佢嚟自受控嘅迴圈、清晰嘅規格、強力嘅驗證、深思熟慮嘅終止，以及一個學識去思考自己思考方式嘅系統。
