我们一起走一遍 VA Agent Swarm 的完整界面设计。这套布局,就是把一个由超过一百个专门 agent 组成的系统,收纳进一个易于理解的工作空间里。它涵盖了系统 composition diagram 中的每一个操作,也带你从最初的一个念头,一路走到一条制作完成、交付完毕的视频。而由于这个平台支持十种不同的工作流程,从 A 排到 J,所以这套设计要能够根据每一种而变化。

我们就从背后的理念讲起,因为每一个决定都是从几条核心原则衍生出来的。

第一条原则是 brief 优先。每一条制作都从一份人工撰写的 brief 开始,也就是有人真正想创作的描述。所以界面就把这份 brief 放在重心位置,输入它就是其他一切自然的起点。

第二条原则是渐进式披露。上百个 agent 一起运作,如果一次性全部显示出来就会让人喘不过气。所以这套设计只会显示当前阶段真正需要的内容,其余的就收起来,等到有需要时才出现。

第三条原则是让 agent 图实时可见。Composition diagram 并不是一幅静止的图,它是实时运行的。用户要看一眼就立刻明白每个 agent 现在在做什么。

第四条原则是由 gate 驱动的信心。在关键时刻,一个 gatekeeper agent 会判断这条制作是否准备好从一个阶段进入下一个阶段。这些转换在界面上会以清晰、明确的批准时刻呈现,让人可以有信心地签核。

第五条原则是 critique 透明度。Agent 之间不断互相评价彼此的成果,而每一条这类消息都可以查看、可以搜索,也都是你可以采取行动的对象。

第六条原则是根据制作类型而变化。十种工作流程模板决定了哪些 agent 会被唤醒、哪些面板会出现,所以界面会根据你在做什么而重新塑造自己。

那么这一切又是给谁用的呢?主要有四类用户。创作者想快速开始一条制作、查看成果、批准各个 gate。制片人则盯紧预算和进度、处理升级的问题、管理团队。技术操作员调试 prompt、检查 agent 的日志、管理模型如何分配。而审阅者或者客户则查看交付物、留下意见,给出最终批准。

我们看看整个应用程序是怎么组织的。最顶部是 dashboard,也就是你的大本营,有一个进行中制作的网格、一个用于新 brief 的快速开始向导,以及一条显示整体系统健康状况的横幅。从那里你会进入 Brief Studio,在那里选模板,以结构化和自由发挥两种方式写 brief,上传参考素材,例如情绪板和剧本,然后确认启动,把一切交给 planner agent。

真正的主力是 Production Console,每一条制作都有一个。它里面有 DAG canvas,也就是实时运行的 composition diagram,显示各个 agent 节点处于不同状态、artifact 从一个 agent 交到另一个的流动,以及成果获得批准的 gate 检查点。还有一个 timeline 视图,以阶段泳道从前期制作一路走到制作、后期和交付,还有里程碑标记和预算消耗叠层。还有一个 agent inspector,让你深入查看任何一个 agent,显示它的身份和角色、它当前的任务和进度、它的输入和输出 artifact、它发出和收到的 critique、它相对于阈值的质量指标,以及它的工具调用记录。还有一个 artifact 画廊,带有版本历史、视频音频图像和文本的预览、来源链,以及并排比较模式。还有一个 critique feed,显示按时间顺序排列的消息流,带有筛选功能和一个供人工介入的位置。还有一个 gate 控制面板,放着待批准的队列、标准清单,批准、拒绝和要求修改的操作,以及来源签核。

在 console 之外,还有一个 agent registry,列出所有 agent,可以搜索和筛选,每个都有详情卡、依赖关系图和性能基准。还有一个记忆和知识区,放着项目记忆、一份反思的情节日志、一本用来构建世界观的系列圣经,以及一个品牌素材库。还有一个交付中心,用来构建母版包、制作各渠道专属的变体、跟踪质量管控状况、监控发行,以及查看发布后的分析。还有一个设置和管理区,处理模型分配规则、成本和延迟预算、API 密钥管理、团队权限,以及合规设置。最后还有帮助和文档,包括一份 agent 词汇表、一份工作流程模板指南,以及一份 API 参考。

现在讲讲 master shell,也就是收纳着一切的外框。横跨顶部的是一条大约六十四像素高的横条,放着标志、一个你用 command-K 就能触发的全局搜索,可以跨 agent、artifact 和 critique 搜索,还有通知、用户菜单和设置。左侧是一条窄窄的导航栏,默认只有图标,当你把鼠标移上去时就会展开显示标签。顶部横条下面是一条情境栏,充当当前制作的面包屑,显示它的名称、阶段、预算和健康状况。画面大部分都是主视图区,也就是当前的工作空间。而从底部向上滑出的,是一个详情抽屉,最多可以占半个画面,让你可以查看一个 agent 或者预览一件 artifact 而不必离开原来的位置。在最底部的是一条细窄的状态栏,一眼就能看到实时遥测数据,例如有多少个 agent 在运行、当前的阶段、相对于上限已经用了多少预算,以及估计还要多久。

导航是分层运作的。最高一层,你用侧边导航栏在各个应用程序区块之间移动。在一个区块里面,你用一条标签栏切换视图,在 DAG canvas、timeline、画廊和 critique feed 之间选择。再深入一些,详情会在抽屉或者弹窗里面打开,例如 agent inspector 或者 gate 批准对话框。而最深的那一层,就是通过右键菜单或者命令面板的情境操作,例如重试一个 agent、比较版本,或者导出一件 artifact。

界面里面每一个界面都直接对应到 composition diagram 中的操作。Brief 向导是入口,把用户连到 planner agent。模板选择器在十种工作流程类型之间选择。DAG canvas 呈现从 planner 到 orchestrator 到 router,再到各 craft agent 的完整流程。一个个节点卡显示每个 agent 的状态。Gate 批准对话框处理由 gatekeeper 和 judge agent 驱动的阶段转换。Critique feed 承载着所有 agent 共用的消息总线。记忆面板服务 memory agent。Agent inspector 处理深入钻研。Artifact 画廊和查看器显示并比较五十二个 craft agent 的成果。还有界面负责 timeline、预算跟踪器、router 设置、一个 prompt 实验室、一个质量仪表盘、一个交付封装器、一个分析面板、一个合规检查器,以及专门给创意、研究和优化 meta-agent 用的面板。还有一个通知中心、一个团队和权限区,以及一个供长期情节记忆用的系列圣经编辑器。

现在逐页逐页地走一遍。

Dashboard 一开始就有一个快速开始区,包括一个新建制作的按钮和一句邀请你描述想创作什么的提示,旁边是一行从 A 到 J 的模板卡,例如爆款钩子、UGC 广告、解说视频、生日视频、短片、企业视频和音乐视频。下面是一个进行中制作的网格,每张卡显示项目名称、它的工作流程类型、当前阶段、一条进度条、完成百分比,以及到目前为止花了多少钱。最底部是一条系统健康状况带,显示有多少 agent 在线、有多少 gate 待处理,以及有多少警示生效中。点击一张模板卡会打开预先加载了那个工作流程的 Brief Studio。点击一张制作卡会打开它的 console。而从零开始就会打开一份空白的 brief。

Brief Studio 会引导你完成几个步骤。首先你选一个模板,studio 会预览那个模板会激活哪些 agent。然后你填写 brief 详情:一个标题、一段两三句自由发挥的愿景陈述、类型、用一条从十五秒延伸到两小时的滑块选时长、宽高比、以一组标签表示的基调、目标受众、一个预算上限,以及一个截止日期。接着你加参考素材,拖入剧本、情绪图像、参考视频和音频,或者从品牌库里拉取素材,你也可以粘贴风格参考链接或者选配音和演员偏好。之后你设置约束和合规,有一些开关处理例如来源签署、无障碍标准、AI 同意核实、数据处理规则,以及你针对哪些平台。最后你审阅和启动,studio 会显示一个计划预览,估计 planner 会创建多少个阶段、会涉及多少 agent、预期成本和预期时间。从那里你可以保存草稿,或者启动制作。

每个模板都有自己的预设。爆款钩子模板默认是竖屏短视频,针对 TikTok 和 Reels,并激活钩子、趋势、社交和留存 agent。UGC 广告模板聚焦于效果目标,引入品牌、文案和 A/B 测试 agent。动画解说视频偏向教育,带有动画、动态图形和旁白。个性化生日模板加入个性化变量、模板、虚拟人和配音。AI 短片解锁完整的创意团队,加上摄影、剪辑和声音。企业培训模板加入学习管理封装和评估。音乐视频带来节拍同步和编舞。虚拟人模板加入声音克隆和对口型。纪录片模板带来档案研究和事实核查。而故事长片模板就激活整个 agent 阵容,每一个 gate 都会运作。

Console 的核心是 DAG canvas。在这里,实时运行的 composition diagram 会变成一个你可以探索的交互图。你会看到顶部的 brief,向下流向 planner agent,然后是 orchestrator,它分支到 router 和 memory agent,再从那里到各 craft agent,例如导演、摄影指导、剪辑师、作曲家和视觉效果总监。它们下面坐着 gatekeeper,它在等待批准,以及 judge,它正忙着评分。每个节点都带有清晰的状态。一个对勾代表完成,显示绿色。一个实心点代表运行中,会闪动蓝色。一个空心点代表空闲或者排队中,显示灰色。一个警告符号代表受阻或者等待批准,显示琥珀色。而一个叉代表失败,显示红色。你可以点击一个节点,在抽屉里打开它的 inspector;双击就会有全屏视图;点击一条边就能看到正在传递的 artifact;点击 gatekeeper 就打开它的批准对话框;或者右键就有情境菜单,带有重试、跳过、检查和查看 critique 等选项。你拖动就可以平移,滚动就可以缩放,按住 control 就可以一次选中多个节点。还有筛选,可以显示所有 agent、只显示正在运行的、只显示受阻的,或者只显示已完成的,加上图层开关,分别控制 orchestration、craft、创意、研究和优化 meta-agent,以及 critique 图层。

Timeline 视图沿着一条横向时间轴讲述同一个故事。每个阶段都有自己的泳道:前期制作、制作、后期制作和交付,有填充的长条显示进度,也有标签标示每个阶段里面的步骤。Gate 标记坐在底部,显示哪些 gate 已经通过、哪些在审阅中、哪些还未处理。而一条预算消耗线就描绘出开支如何随着制作向前推进而上升。

Artifact 画廊把所有 agent 制作出来的东西以网格或者列表卡片呈现。每张卡显示一个缩略图,可能是视频帧、音频波形、图像,或者文件图标,连同一个版本徽章和它的状态、制作它的 agent、一个关键质量指标,以及一个确认它已经签署的来源徽章。点击一张卡会在抽屉里打开 artifact 查看器,在那里你可以并排比较版本、浏览完整版本历史,以及追溯完整的来源链。

Critique feed 是 agent 之间一场鲜活的对话。每一条记录显示时间、谁在对谁说话,以及一个严重程度,从信息性到警告到严重不等。举个例子,一个剪辑 agent 可能会告诉导演某场戏的节奏拖得稍微长了一点,建议剪掉一些片段,并附上一条节奏曲线作为证据。一个质量 agent 可能会标出某几个帧出现手部瑕疵,建议用新的 seed 重新生成。一个合规 agent 可能会警告所有人说一个声音克隆的同意即将过期,并在续期之前封锁交付。而 judge 可能会宣布一场创意辩论已经解决,解释哪一方获胜,以及赢了多少分。每条消息都附带操作,例如接受、拒绝、讨论、自动修复,或者升级给人处理。而在底部坐着一个人工介入的位置,你可以在那里向任何 agent 输入你自己的 critique 或者指示、直接提及它们,并设置优先级。

当一个阶段准备好推进,gate 批准对话框就会出现。它会讲明是哪个 gate、是哪个转换,显示 gatekeeper 的评估和 judge 的数字分数,并列出一份标准清单,每一项都标示为通过或者需要注意,例如剧本覆盖、对白独特性、分镜完整度、预算在上限之内、风格一致性,以及一个合规预检。它列出待审阅的 artifact、提供一个留言框,并给你三个核心操作:批准、拒绝,或者要求修改。它也会确认来源签署,把这次签核归于你。

Agent inspector 就是你深入了解一个 agent 的地方。它显示它的身份,包括它的类别和它遵循的推理模式、它从多少个 agent 接收工作,以及它对多少个 agent 作出评价。它显示它当前的任务和状态,包括它在做第几次自我精修。它以长条形式显示质量指标,对照着阈值,例如文本对图像的吻合度、节拍覆盖和节奏吻合。它列出输入和输出 artifact。它记录工具调用,例如一个向视频生成模型发出的请求,或者一次记忆调取。而它也显示 critique 总线,收到和发出的都有,让你看到它和其他 agent 之间的一来一往。从这里你可以重试任务、跳过它、发出 critique,或者查看完整历史。

Agent registry 是系统里面每一个 agent 的目录,分门别类,例如线上创作层、摄影、剪辑、声音、表演、发行、教育、AI 专家,以及各种 meta-agent 组别,分别负责 orchestration、创意、研究和优化。每一行显示 agent 的编号、它的名称、它的推理模式、它使用的工具,以及它当前的状态。点击任何一行就会打开一张显示它完整能力的详情卡。

记忆和知识面板放着系统的长期理解。项目记忆可以用语义和精确两种方式搜索,并显示最近的条目,例如关于某场戏基调的笔记、某个角色的服装规则、一次预算修订,或者一个锁定的风格 seed,连同哪些 agent 一直在访问它们、访问了多少次。还有给情节日志、系列圣经、品牌库,以及一个世界观数据库的板块。

交付中心管理把完成的作品送出去这件事。一个渠道矩阵列出每个出口,例如 YouTube、TikTok、Meta、广播、影院和档案,并就每一个显示格式、质量管控状况、字幕、无障碍级别、来源,以及一个整体已准备好或者待处理的状态。一份质量管控摘要把检查分成三个级别:技术层面,例如响度、色彩和分辨率;创意层面,例如节奏、节拍同步和风格;以及合规。从这里你可以封装所有已准备好的东西、修复待处理的问题,或者打开完整的质量报告。

设置里面的 router 设置,就是你决定哪个模型处理哪些任务的地方。一个路由规则表把每种任务类型,例如视频生成、声音合成、虚拟人渲染、图像生成、创意语言模型、judge 模型和音乐生成,跟一个主模型、一个后备,以及每项任务的最高成本配对在一起。下面是成本防护:一个全局预算上限、一个警示阈值、一个质量会自动降级的点,以及一个需要人工覆写的硬性停止点。

Prompt 实验室是用来制作和精炼生成 prompt 的工作空间。你选一个 agent、一个模型和一个镜头,然后写一个 prompt 描述你想要的场景,并设置参数,例如 seed、引导、步数和负面词。你可以生成、跑一次优化、做 A/B 测试,或者逐个 seed 走一遍。一个优化历史表跟踪每一次迭代、它的分数、改了什么的摘要,以及它有没有被接受,让你看到这个 prompt 随时间如何改善。而一行生成出来的成果,就让你比较各次迭代,并把最好的那个送给导演批准。

质量仪表盘把所有评估分数汇总起来。它以长条形式显示整体指标,对照着它们的阈值,涵盖例如视频基准分数、文本吻合度、动态质量、美学、音频清晰度和响度。它把结果逐个镜头分拆,标出手部、面部识别、时间一致性和风格。并浮现回退警示,精准指出哪里分数掉了,有选项可以全部自动修复、人工审阅,或者忽略那些轻微的。

现在追踪一次完整的制作启动流程,也就是从念头到成片的旅程。它从 dashboard 开始,在那里你可以点击一个模板、从零开始新建一条制作,或者输入全局搜索,让系统建议一个模板。接着是 Brief Studio,在那里你确认模板、填写 brief、上传参考素材、设置约束,并审阅计划预览。然后你启动:brief 去 planner agent,它把它拆成一个分阶段的图;planner 交给 orchestrator,它初始化执行;orchestrator 调用 router 去分配模型和 agent;memory agent 用 brief 和参考素材进行初始化;而界面就滑入 production console。

从那里,这条制作会经历它的各个阶段,大部分自动进行,但在各个 gate 都有人处在循环之中。在前期制作,agent 写剧本、画分镜、开发外观、选配音,并勾勒初步的音乐主题,而创意和研究 meta-agent 就喂入想法和情境。Gatekeeper 检查第一级标准,一个 gate 对话框出现,你就批准。在制作阶段,导演发出镜头意图,prompt 工程师制作 prompt,router 把它们送去视频模型,摄影指导核实构图,而质量 agent 就逐帧检查,优化 agent 就一边走一边调试。Critique 消息流动,artifact 在画廊出现,你可以通过 critique feed 介入。又一个 gate,又一次批准。在后期制作,剪辑师组装剪辑,调色师套用色调,声音团队铺入音频并做混音,视觉效果总监做合成,而一个无障碍 agent 就加入字幕和口述影像。Judge 对照一个评分标准评分,gatekeeper 检查最后的标准,你又再批准。然后就到交付,中心显示渠道矩阵,发行方就每个出口封装,合规给一个最终法律签核,来源在所有东西上面签署,你就批准发行。而可选地,在发布之后,一个分析师收集效果数据,优化 agent 研究留存和广告支出回报率,分析面板显示结果,而那些学到的东西就反馈进记忆,供未来的制作使用。

不同模板会以一些显著的方式改变体验。爆款钩子把一切压缩在短短几分钟之内,把趋势情报放在最前最中,并预览留存曲线。UGC 广告突出效果目标,并显示广告支出回报的预测。动画解说视频加入一个学习目标编辑器和一个评估构建器。个性化生日加入一个变量模板编辑器和一个批量渲染队列。短片显示完整的图,五十二个 craft agent 全部激活。企业培训加入学习管理封装和学员模拟。音乐视频加入一个节拍网格和波形叠层。虚拟人模板加入一个虚拟人设计工作室和一个同意流程。纪录片加入一个档案研究面板和来源评级跟踪。而故事长片就加入集数和幕次导航、系列圣经、完整阵容,甚至电影节策略。

为了确保系统里面没有东西被遗漏,composition diagram 中每一个节点和每一条边都对应到界面里面具体的界面。Brief 对应到向导和模板选择器。Planner 出现在 canvas、timeline 和 inspector。Orchestrator 出现在 canvas、timeline、通知中心和预算跟踪器。Router 出现在 canvas、router 设置和优化面板。五十二个 craft agent 出现在 canvas、它们的节点卡、inspector、画廊、查看器、prompt 实验室和质量仪表盘。Critique 消息流经 feed,并进入每个 inspector 的 critique 标签。Judge 和 gatekeeper 出现在 canvas 和 gate 对话框,gatekeeper 还连着合规。Memory agent 对应到记忆面板和系列圣经。而每一族 meta-agent,创意、研究和优化,都有自己专属的面板,带有一些控制,例如头脑风暴卡、节拍表可视化、风格锁、情绪板组合器、趋势时间线、竞争对手网格、来源评级徽章和留存预测器。

讲几句关于响应式和无障碍。这个布局会跨断点适应:在大型桌面所有东西一次性可见;在标准桌面导航栏收缩成图标,抽屉叠在 canvas 上面;在平板就变成单一窗格加标签;而在手机就 dashboard 和 brief studio 保持可用,而 production console 就变成只读,用来监控。无障碍是认真对待的,目标是达到最新网页内容指南的双 A 级别。文本符合严格的对比度,状态除了用颜色也用形状传达,整个东西都可以用键盘导航,有可见的焦点环和命令面板,屏幕阅读器会得到适当的地标和实时区域,会播报 agent 状态的变化,也有一个减少动态的模式,把动画变成静态转换。各个节点状态刻意用不同的形状,所以它们从来都不是只靠颜色,这一点对色盲用户来说仍然清晰可读。深色和浅色主题都支持,而 canvas 在两种模式下都用一个柔和的背景,让节点保持可见。

这套设计建立在一个可复用组件的库之上。有 agent 节点卡、artifact 卡、critique 消息、gate 检查点、指标长条、timeline 泳道、brief 字段、抽屉面板、一个命令面板、通知徽章、来源徽章和预算量规,每个都有自己一套变体。而这些会组合成复合模式:agent inspector、gate 对话框、prompt 编辑器、渠道行和 timeline 阶段,每一个都是由那些较小的部件拼成。

交互模式把一切串联在一起。命令面板,用 command-K 触发,是一个在任何地方都能到达的全局搜索和操作启动器,显示最近的项目、常用操作,以及一个实时筛选的 agent 列表。人处在循环之中的时刻处理得很优雅:当到达一个检查点,gate 对话框就出现;当一个 agent 需要输入,critique feed 就会高亮;当触及预算阈值,一条横幅就警告;当有合规封锁,一个弹窗就浮现;而质量仪表盘就标出回退。实时更新让一切保持最新:agent 状态每隔几秒平顺过渡,critique 消息通过 websocket 滑入,artifact 完成时以动画滑入画廊,预算量规逐项任务更新,而质量长条就随每次评估移动。而批量操作让你一次性重试所有失败的 agent、批准所有符合阈值的待处理 gate、导出选定或者全部 artifact、比较任何两个版本,或者清除过时的记忆。

还有一个通知和警示系统,有清晰的优先级。严重警示,例如一个合规封锁或者预算超支,会占据整个画面,连同一声提示音,而且不采取行动就无法关闭。高优先警示,例如一个 gate 准备好批准,会以一个带徽章的吐司显示,并要求在几分钟之内采取行动。中等警示,例如一项完成的任务或者一条新 critique,只是让一个徽章递增。而低优先警示,例如一个优化建议,就只显示一个徽章。通知中心把它们全部收集起来,把未读和已读分开。

在底层,这个界面为每一条制作维持一个结构化的状态,通过 websocket 同后端保持同步。一个制作对象放着它的身份、它的模板、它当前的阶段、brief、agent 图、预算、各个 gate、artifact、critique、记忆、交付状态,以及可选的分析。这个图本身跟踪节点和边、当前阶段和一个完成百分比。每个 agent 节点带有它的身份、它的状态、它当前的任务、它的指标、它发出和收到的 critique、它的工具调用,以及它的 artifact。Gate 跟踪它们的标准、judge 的分数、它们的状态、谁批准了它们,以及它们有没有被签署。Artifact 跟踪它们的版本、类型、制作的 agent、质量分数、来源状态和父素材。而 critique 消息跟踪发送者、接收者、严重程度、内容、任何附件、状态,以及任何人工回应。

至于技术,这套设计建议用一个现代的网页技术栈:用 React 和 Next.js 做框架,把给 dashboard 用的服务器渲染和给实时 console 用的客户端交互融合在一起;以 websocket 同步的轻量状态管理;一个专为渲染 DAG 而设的节点图库;给 timeline 用的自定义可视化;给视频和音频预览用的强劲播放器;给一切实时更新用的 websocket;一个无障碍、以功能为先的样式方案;给指标和分析用的图表库;一个经得起考验的命令面板组件;一套干净的图标;以及在 prompt 实验室里面一个完整的代码编辑器,用来编辑 prompt 并带有语法高亮。

最后,这套设计指向一些配套的线框文件:一个用于完整应用程序外壳,一个把 composition diagram 对应到界面界面上。合起来看,这套设计把一个极其复杂的 agent 群体,变成一个人类真正可以理解、引导和信任的工作空间,从最初那份 brief,一路走到完成、交付的影片。
