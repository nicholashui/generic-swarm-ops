我们来聊聊项目是怎么运作的，以及为什么它会位于实际的制作之上多出一层。这里的核心思路，就是给你一个层面，让你可以在任何 agent 启动之前、在花掉一分钱之前，先去规划、反复修改，并和团队一起协作。

先说最基本的区别。项目是一个容器。创建是免费的，会长期保存，也是为协作而设计的。而制作则是一次执行。这部分才真正需要付费，会运行 agent，也会产出真正的成品。

想象有个项目叫做「Brand Campaign Q3」。创建它免费，保存它免费，在里面做规划也免费。在这个项目里，你可能会放共享素材，比如品牌套件、声音和风格参考。你有一个团队，由一位所有者加上编辑和审核者组成。还有一个预算池，比如预留了二百四十美元。也有一些默认设置，处理合规、模型和平台之类的内容。然后就到制作本身了。其中一个可能是已完成的主打视频，花了六十二美元。另一个可能是正在运行的 TikTok 剪辑版，到目前为止用了二十八美元。而第三个可能还停在草稿阶段的培训内容，暂时一分钱都没花。

这里有一条规则把所有东西串联在一起。在你明确点下启动之前，是不会花掉一分钱的。在那一刻之前发生的所有事情，都是完全免费的准备工作。

那么从零到一个运行中的制作，整个流程是什么样的呢？首先你创建一个项目，瞬间完成而且免费。然后你做准备，收集素材、邀请团队、选设置，这个过程可以花几天甚至几周，仍然是零成本。接着你创建一个制作草稿，brief 会保存下来，也可以继续修改，同样免费。之后你拿到一个成本估算，是由规划器提供的预览，也是免费的。然后就到启动了，这一刻才开始花钱，agent 也开始运作。从那时起，你一边监控进度、一边审批，制作就会运行。最后就交付完成。当你准备好做下一个，只需要从草稿那一步再重复一次。

现在我们一个个界面来看。仪表盘就是你的起点。最上面有个区域叫做「我的项目」，加上一个创建新项目的按钮。每个项目都会以卡片形式出现。Brand Q3 可能显示三个制作，其中两个正在运行、用了九十美元，几分钟前更新过。旁边有个叫 Luna Short Film 的项目，显示一个已完成的制作、九十五美元，三天前更新过。而且永远都有一张卡片让你开始一个全新的项目。

下面你会看到一些活跃中的制作，是从你所有项目汇集而成的，这样你就能一眼看完所有正在进行的内容，每个都附带一条小小的进度条。再往后还有一个快速开始的区域，是给你只想快速做点东西、不想做设置时用的。你选一个模板，系统就会在后台帮你处理好这个项目。模板涵盖了钩子、用户自制内容、解说片、生日视频、电影等等。

当你真的创建一个新项目时，会弹出一个对话框。你给它起个名字。你可以加一段选填的描述，比如说一个横跨社交媒体和网页的 Q3 品牌认知活动。然后就到预算池，你在这里设定里面所有制作的总预算，或者你可以选择无上限，纯粹用多少付多少。你还会选一个付款方式，比如一张已存档的信用卡。

团队那部分是选填的，你随时可以以后再加人。你默认是所有者，你可以用邮箱邀请其他人，并分配角色。这些角色值得了解一下。所有者有完整的控制权，包括付款和删除。编辑可以创建和启动制作，并管理素材。审核者可以查看、评论和审批关卡。而查看者只有只读权限。

还有一个默认设置的部分，会应用到每个制作，除非你逐个覆盖。你可以设定合规选项，比如 C2PA、无障碍标准、演员协议或者隐私法规。你选一个模型偏好，看你想要成本优先、质量优先还是速度优先。你也可以现在就连接品牌套件，或者以后再加。底部有一个友好的提示：创建项目是免费的，只有当你启动制作时才会收费。

一旦你进入这个项目，就会来到项目工作区。最上面是项目名称，附有归档它或者打开设置的选项。还有一行标签页：概览、制作、素材、团队、设置和活动。

在概览标签页，你会一眼看到一组状态卡。其中一张显示你的制作，可能有三个，一个完成、一个正在运行、一个在草稿。另一张显示预算，二百四十美元里用了九十美元，大约百分之三十八。第三张显示团队，可能有三位成员。第四张显示你的素材，比如十二个文件。

卡片下面就是制作列表本身，附有一个开始新制作的按钮，再加上排序和筛选。每个制作都列得很清楚。主打视频标注为完成，是一个 Type E 的作品，两天前完成，花了六十二美元，片长五分二十秒，已交付到 YouTube 和 TikTok，还有选项可以查看成品、看分析、或者复制成草稿。TikTok 剪辑版正在运行，是一个 Type A 的作品，正处于制作阶段，用了二十八美元，完成了百分之五十五，大约还差两分钟，你可以打开它的控制台或者暂停它。培训单元还是草稿，是一个 Type F 的作品，还没启动，零成本，brief 完成了百分之八十，估算成本大约三十五美元，提供按钮去编辑 brief、获取估算、启动或者删除。

再往下就是最近活动的动态。它可能会记录几分钟前编辑 agent 在 TikTok 那个作品完成了粗剪，五分钟前你审批了它的第一个关卡，两天前主打视频交付到了 YouTube 和 TikTok，三天前 Sarah 以审核者身份加入。

素材标签页就是你的共享素材库。最上面有按钮去上传素材或者连接品牌套件。品牌套件那部分放着你的标志、字体、颜色定义、一份指南和一份语调文件，而这些全都会自动加载到每个制作的品牌 agent 里。然后还有一个声音库。你可能有一个充满活力又年轻的品牌声音，通过声音服务克隆出来。另一个是权威又成熟的，专门训练出来的。还有一个中性、清晰的旁白，使用标准的文字转语音。你随时都可以添加更多声音。

还有一个风格参考的部分，放着比如一张被形容为新黑色电影风格的情绪图、一段节奏参考视频，和一张构图参考。最后就到剧本和文档的区域，放着比如品牌信息指南、目标受众研究和竞品分析。这里的重点是，这个素材库里的所有东西，都会自动提供给项目里的每一个制作使用，而那些 agent 引用它们时，不需要任何人重新上传。

接着就到制作草稿界面，在这里你可以在启动之前编辑所有东西。它的核心是 brief 编辑器。你会看到选好的模板，这次是企业培训，你可以更改。你给这个制作起个标题，比如「培训单元：产品入门」。然后就到愿景，一段文字描述，比如一段十分钟的互动培训视频，教新员工怎么使用我们的产品仪表盘，效果应该现代、亲切，还要包含知识测验。

从那里你设定细节。片长十分钟、宽屏格式、语调亲切又专业。你列出学习目标，比如浏览仪表盘和创建报告。你选一个评估方式，可能是每节之后一个测验。你还选交付格式，比如常见的在线学习打包标准或者独立视频。你可以从项目里拉入参考素材，比如品牌套件和一个指定的品牌声音，你也可以添加自己的额外内容，比如一个产品截图的压缩包。你给这个制作设定一个预算，比如三十五美元，你也看到预算池二百四十美元里还剩一百五十。所有东西都会自动保存，所以它会说几分钟前最后保存过。

然后就到成本估算预览。你随时可以刷新它，它也会显示上次什么时候计算的。它把成本逐项拆开：剧本和教学设计几美元、渲染那段十分钟的虚拟主播大约十二美元、声音合成四美元、动态图形六美元、生成评估两美元、质量管控和合规三美元、打包到学习系统一美元。加起来总估算大约三十一美元，上下浮动百分之十五左右。它还告诉你需要多少个 agent，大约十八个，以及估算的时长，大概八分钟。

还有一个部分是团队对草稿的评论。审核者 Sarah 可能问你能不能加一节讲报告导出，而你回复说这个主意很好，已经加到了学习目标里。任何人都可以参与发言。

接着就到操作了。你可以保存草稿、分享给人审核，或者启动制作。启动按钮对它会做什么讲得很清楚。它会从项目预算池里扣取最多三十五美元。它会启用十八个 agent，涵盖教学设计、虚拟主播、声音、学习系统等等。它开始制作，而你会在制作控制台监控它。它估算大约八分钟完成。

当你点下启动，会弹出一个确认对话框，让你没有意外。它会列出制作、模板和项目。它给一个成本摘要：估算三十一美元、预算上限三十五美元、项目剩余预算一百五十美元、这次启动之后大约剩一百一十九美元。它清楚说明会发生什么：规划器拆解你的 brief、十八个 agent 启用、你会在控制台看着进度、关卡审批会暂停等你查看、估算大约八分钟完成。你勾选一个方框，确认你明白这会动用你的项目预算，然后就启动。

退一步，看看整个信息架构是怎么搭在一起的。最顶层是仪表盘，它容纳你的项目网格、横跨所有内容的活跃制作，以及快速开始。然后就到项目工作区，这是新的，容纳概览、制作列表、共享素材、团队、设置和活动记录。还有制作草稿，也是新的，容纳 brief 编辑器、成本估算、团队评论和启动按钮。启动之后你就会进入制作控制台，这个本身已经存在，有它的 agent 图谱画布、时间轴视图、成品画廊、评论动态、关卡控制和 agent 检查器。围绕这一切的，就是本身已有的区域：agent 注册表、记忆与知识、交付中心、设置与管理，以及帮助与文档。

在后端，项目层值得一提，因为它完全没有 agent 参与。它纯粹是创建、读取、更新和删除的操作。你可以创建和列出项目，获取或者更新单个项目，以及归档它。你可以上传、列出和移除共享素材。你可以邀请成员、更改他们的角色，以及移除他们。至于制作，你可以免费创建草稿、列出它们、在还是草稿时编辑 brief、预览成本估算，然后启动，这就是 agent 真正开始的唯一一个点。你也可以暂停一个运行中的制作、恢复一个暂停的、把一个制作复制成新草稿，或者删除一个草稿。

了解一下数据的形态也有用，至少在概念上。一个项目带着一个身份和名称、一段选填的描述、一位所有者，以及一个状态，不是活跃就是已归档。它追踪它的预算池，这个可以是空值，也就是没有上限，连同已经花了多少和付款方式。它容纳它的成员，每位都有一个用户、一个角色，和加入的日期。它容纳共享素材，每件都标注类型，是品牌套件、声音、风格参考、文档还是剧本，附有名称、位置和一些元数据。它带着合规、模型策略和平台目标的默认设置。它还记录它什么时候创建和最后更新。

至于一个制作，它就属于某个项目。它有一个标题和一个模板，以 A 到 J 的字母标识。它会经历一连串状态：草稿、启动中、运行中、暂停、已完成或者失败。它容纳一个 brief，在还是草稿时可以继续修改，捕捉愿景、一个选填的类型、片长、宽高比、语调、目标受众、按素材的参考、任何限制、平台目标，以及模板特定的细节比如学习目标或者节拍数。它追踪一个预算上限和已花的金额。它可以容纳一个由规划器预览来的估算，包括低、中、高三个成本数字、需要的 agent 数量、一个估算时长、按类别的拆解，和它什么时候生成。它保留团队对草稿的评论。而只有在启动之后，它才会多出比如 agent 图谱、成品，以及启动和完成的时间戳。

最后，为了没耐心的用户，有一条快速开始的路径，一样走得很快。从仪表盘你点一个模板，比如一个爆款钩子。系统会自动帮你创建一个未命名的项目，你以后可以改名，以及一个已经预先填好 brief 的制作，再把你直接放到草稿模式的 brief 编辑器。你填好最少的信息，点下启动，那些 agent 就开始了。这个项目仍然在后台静静地存在，所以你以后可以添加更多制作、给它改名和邀请团队，但最初的阻力几乎是零。

那么总结一下这个项目层到底改变了什么。以前，仪表盘会直接带你进入一个 brief 工作室，然后立即启动。现在则会流经一个项目、一个草稿、一个编辑阶段，并只在你准备好时才启动。以前，每个制作都是独立存在的。现在制作就住在项目里面。以前，素材是逐个制作分开上传的。现在则有一个供整个项目共享的素材库。以前，在启动之前没有团队协作。现在你的团队会审核草稿并留下评论。以前，在投入之前你对成本没有可见性。现在你在任何草稿上都能拿到一个免费估算。以前，你没办法在不花钱的情况下反复修改 brief。现在草稿模式就意味着无限次免费修改。以前，没办法把相关的视频归组。现在一个项目就把一个活动的所有变体归在一起。以前，预算只能按制作计算。现在则有一个横跨整个项目的预算池，再加上逐个制作的上限。
