我们这次聊聊，你到底怎么管理系统里一个一个独立的 agent。不是讲大型的制作流程，而是 agent 本身。你怎么查看它是怎么设置的？怎么单独地去测试它？又怎么让它随着时间变得越来越聪明？这些就是 Agent 管理区域要做的事，它提供了三种不同的工作方式。

第一种是配置，也就是 Configure。第二种是 Playground，游乐场。第三种是 Knowledge，知识。可以这样理解。配置是你查看和编辑一个 agent 怎么构建的地方。Playground 是你单独测试一个 agent 的地方，你给它一个任务，再看着它怎么思考。而知识就是你喂新资料给它，让它表现得更好的地方。好处在于，这些全都不需要跑一次完整制作，而且除非你真的开动 Playground 去生成东西，否则完全不用花钱。

我们先从配置开始讲。你可以在 Agent 注册表里进入任何一个 agent，点进去，就会来到它的配置界面。我们就用 DirectorAgent，也就是导演 agent 来做例子。界面顶部你会看到几个标签页，分别是配置、Playground、知识、历史和指标，所以跟同一个 agent 有关的所有东西都集中在一个地方。

配置界面分成几个清晰的区块。最上面是身份。也就是 agent 的名称、它的类别，以及一段简短描述讲它做什么。对导演来说，描述说它掌握创意愿景，它发出镜头意图，它设定节奏，也负责批准每个镜头。换句话说，它是整个制作的创意决策者。

下面是系统提示。这个是 agent 的核心，也就是界定它是谁、它怎么行事的指令。导演的提示要它扮演一位顶尖的电影导演，对视觉叙事有深厚的认识，灵感来自像 Criterion 评论音轨、导演工会的研讨会，以及像 Scorsese、Lynch、Gerwig 这些导演的大师班材料。接着就列明它的工作。把剧本场景转化成精准的镜头意图。界定镜头运动、构图和灯光氛围。设定贴合该类型片预期的节奏。对照创意愿景去检视生成出来的镜头。并且向其他 agent 发出创意意图的修订。提示还指明 agent 应该输出什么，比如镜头运动、构图、主体、风格、时长和情绪。你可以看到提示有多长，可以展开编辑器获得更多空间，也可以通过版本历史把以前的版本调出来。

接着是架构模式。这个是 agent 内部怎么推理的方式，你可以在一个菜单里选。这里的导演用的是一种自我精炼模式，也就是 agent 会循环地改进自己的作品，满意了才接受。其他选项还有像 Reflexion、ReAct、宪法式 AI、多 agent 辩论，等等。除此之外，你还可以设定实际的限制，比如在确定下来之前它最多可以循环多少次自我精炼、控制创意程度的温度值，以及它输出的最大长度。

之后是模型分配。每个 agent 都会有一个主模型，以及一个备用模型，以防第一个不可用。在生成方面，你也会分配一个主要的生成工具和一个备用。这样，如果主要的视频生成器太忙或者出问题，agent 也有可以转去的第二个选择。

再之后是工具。这里是一个清单，列明 agent 可以用的功能。导演开了几个视频生成工具，还有调取记忆和存储决定的能力，以及一个用来衡量片段和文字匹配得有多好的评分工具。有两三个工具关掉了，因为导演根本不需要，比如图像生成和语音。而你随时都可以添加自定义工具。

下一个区块是质量评分标准。这里是你界定对这个 agent 来说，「好」究竟是什么的地方，用一组可衡量的阈值来表达。对导演来说，文字和视频的匹配分数要达到一个最低标准。每一个故事节拍都要照顾到，没有例外。节奏要足够贴合该类型片，而视觉风格要在一个设定水平之上，从一个镜头到下一个镜头都保持一致。每个指标都有一个阈值和一段简短描述，你随时可以添加新指标或者调整阈值。

接着是关系，这里就能看出这个 agent 群体的社交结构。你决定这个 agent 接受哪些 agent 的批评，又可以反过来评论哪些 agent。比如导演，它接受编剧、剪辑师和观众模拟器的意见，而它就向剪辑师、摄影指导、编剧和作曲家提出批评。这个就是各个 agent 互相制衡的方式。

接着是成本控制。你可以为单个任务设定最高开支上限，限制同一时间可以运行多少个副本，设定一个超时，让任务不能无止境地跑下去，也可以决定出问题的时候应该重试多少次。

最底下就是操作按钮。你可以保存你的改动、把所有东西还原成默认、导出整个配置，或者复制这个 agent 做一个变体出来。这里有个重要提示。你做的任何改动只会应用在以后的制作上。任何已经在运行的，都会继续用它开始时的配置，这样你就永远不会打乱进行中的工作。

现在我们来到 Playground。这个是整个系统里最有用的工具之一。可以把它当成一个沙盒，你交一个单独的任务给这个 agent，然后看着它思考和生成，完全和任何真实制作隔离开。

界面分成左右两半。左边是输入面板，你在这里设置这个任务。你选一个任务类型，比如生成镜头意图、评论一件作品、检视一个剪辑，或者运行一个自定义提示。然后你给它场景背景。在我们的例子里，场景是室内，一家咖啡店，夜晚，雨水沿着窗户滑落。一个叫 Maya 的角色独自坐着，望着她的手机。她收到的最后一条信息，简单地写着，我不来了。她把手机翻过来，正面朝下放下。你可以放一些参考图进去引导观感。你也可以模拟其他 agent 的批评，这样你就能看到当比如剪辑师或者观众模拟器提出反对意见时，导演会怎么回应。你还可以锁定一个风格或者记忆背景，比如一种跟着某个生成种子的新黑色电影、忧郁的氛围。

下面是运行设置。你可以只为这次测试覆盖模型。你可以选究竟真的生成视频，这个会花真钱，还是只运行纯文字，那样就基本上免费。你可以开启自我精炼，并设定它有多少次迭代。在你确定之前，它会显示一个预估成本。然后你只需要运行这个 agent，或者免费地只运行纯文字，甚至可以把它和另一个 agent 并排比较。

右边是输出面板，而这里就开始有意思了，因为你不是只看到最终结果。你会看到这个 agent 一步一步的思考轨迹。它首先分析场景背景，注意到这个夜晚的设定、情感上的孤立，以及雨水这个母题。然后它伸手进记忆里，把跟这个角色有关的既定视觉风格调出来，结果是新黑色电影、冷色调、浅景深。接着它生成真正的镜头意图。一个从中景慢慢推进到特写的推镜。一个偏离中心的构图，右边摆着一张空椅子，去突出她的孤独。灯光只用现场光源，霓虹灯透过雨水散发出来。一种孤独和认命的情绪，大约运行八秒。之后它调用视频生成器，你会看着它工作时的状态。最后它评估自己的输出，为视频和提示之间的匹配打分。在这个情况下，分数达到了阈值，所以这个 agent 在第一次就通过并接受了结果。

在思考轨迹下面，你会得到真正的输出。生成出来的视频，连同它在匹配、美感和风格一致性方面的分数，还有镜头意图本身，随时可以复制或者导出。还有一个成本和表现的摘要，告诉你总开支、用了多少时间、用了多少 token、运行了多少次迭代、做了多少次工具调用，以及用了哪个模型。从那里你可以把整件事保存成一个测试用例、加入 agent 的知识、再运行多次，或者试另一个模型。

Playground 的深度很够。纯文字模式让你看推理过程，不用为视频付钱。视频模式就实实在在地从头到尾运行一次。模拟批评让你模拟其他 agent 的意见。比较功能让你把同一个输入分别运行进两个不同的 agent，或者同一个 agent 用两套不同的模型设置去运行，这样你就可以用一种 A 对 B 的方式去权衡质量和成本。你可以把任何一次运行加书签做测试用例，留着以后做回归测试。思考轨迹让你看到每一个推理步骤、工具调用和决定。而历史就会保存每一次运行，让你日后可以调出来再跑或者比较结果。

这就带到第三种模式，知识。这里才是你真正让一个 agent 变聪明的地方，方法是喂新的参考、例子、修正和领域专业知识给它。它分成五种知识。

第一种是参考文档。这些文档会嵌入到 agent 的知识库里，让它做事的时候可以从里面提取资料。对导演来说，可能是 Criterion 的评论笔记、导演工会研讨会的文字记录、大师班笔记、构图指南、一份按类型片列出节奏预期的文件，以及任何你自己上传的自定义风格指南。你可以通过上传、粘贴文字，或者从链接导入来添加文档。在背后，这些东西会被拆分成片段、嵌入，然后存储在一个向量数据库里，让这个 agent 在做任务的时候可以只提取相关的部分。

第二种是少样本例子。这些是一对对的输入和输出，用示范来教这个 agent。比如，一个浪漫喜剧的公园场景，输入一段示范场景描述，输出一个理想的黄金镜头意图。或者一个动作飞车场面，应该产生快速的剪辑。又或者一个恐怖片的揭示场景，需要慢慢铺垫、固定镜头，以及最少的剪辑。你可以手动添加例子、从一次 Playground 运行里提取进来，甚至从一份文档生成出来。当任务类型和其中一个例子吻合的时候，它就会被放入提示，而一般来说，你提供的好例子越多，这个 agent 就越稳定。

第三种是修正和反馈。这个是一份记录，把过往的错误和 agent 本来应该怎么做配对起来。其中一条修正可能记下了，在某个制作里，这个 agent 为一个亲密场景用了手持镜头，但其实它应该用一个固定三脚架配慢推镜，因为手持代表的是紧张急切，而不是亲密，而安静的情感场景需要的是静止或者慢移动镜。另一条可能记下了，这个 agent 为竖向的社交内容用了宽银幕构图，但其实它应该用竖向，教训就是选构图之前永远要先看清楚目标平台。修正可以来自你的直接反馈，也可以在一件作品被否决的时候自动检测到。它们会被存储成一种情节记忆，让这个 agent 遇到类似任务的时候，可以查一下自己之前出过什么错。

第四种是自定义规则。这些是 agent 必须遵守的硬性约束，加在它系统提示之上。比如，除非类型片是恐怖或者惊悚，否则永远不要用倾斜构图。每个场景至少要包含一个定场镜头。社交内容的镜头要保持在十二秒以内。又或者，当预算所剩无几的时候，优先用固定镜头，因为这样生成更便宜。这些规则会被当成硬性约束注入，而这个 agent 在每一次自我精炼循环里，都会对照这些规则进行自我检查。

第五种是评估基准。这些是测试用例，让你衡量你的知识改进究竟有没有真的生效。你可能有一套十二个测试用例，连同上次什么时候运行过、通过了多少个的记录。当你加了新知识之后，你再运行一次基准，看看情况有没有好转。每个测试都会列出它的输入、预期的行为、上次的结果，以及通过还是不通过。这样如果一个关于竖向 TikTok 构图的测试还是返回横向，你一眼就能看出这个 agent 还没学会那个教训。

要理解这些东西，关键在于明白当这个 agent 在做一个真实任务的时候，所有这些知识实际上怎么流入它里面。模型收到的提示，是由一层层拼出来的。首先，是来自配置的系统提示。然后是自定义规则，明明白白地列出 agent 必须遵守的东西。然后是从参考文档里提取出来、和手头任务吻合的相关知识，所以对一个亲密、情感丰富的场景来说，它可能会提取一段笔记，讲某些导演怎么用固定画面去承载情感的重量。然后是和任务类型吻合的相关少样本例子。然后是修正记忆，提醒这个 agent 要避开过往的错误。最后是来自当前制作的任务背景，包括场景本身和其他 agent 的任何批评。所有这些层次会合并成一个提示，送去模型，而这个 agent 就带着这个完整的画面去回应。

那么，随着时间去改进一个 agent，实际上会怎么发展呢？它遵循一个自然的循环。首先你在制作期间观察到一个问题，比如导演不停为安静场景用手持镜头，或者一次又一次通不过同一个质量标准。然后你在知识标签页加一条修正，记下它做错了什么、应该怎么做、以及为什么。接着你在 Playground 测试它，给它一个类似的场景，看看它现在是不是表现正确。如果是，那就很好。如果不是，你可能需要一条更强的规则或者更多例子。所以如果只靠修正还不够，你就加一个少样本例子，为那个确切情况提供一对黄金的输入和输出。然后你运行基准，确保你真的改进了，而没有不小心搞坏了其他东西，看着分数往上爬。最后它会自动部署，所以下一个制作就会用改进后的 agent，而任何已经在运行的制作就会保持不变，维持稳定。

在存储方面，所有这些知识都是按每个 agent 来保存，而且可以是全局的，也可以绑定在某一个项目上。对每个 agent 来说，系统会追踪它的参考文档，拆分成片段并嵌入以供提取。它会按任务类型追踪少样本例子。它会追踪那些修正，每一条都连着背景、错误行为、正确行为、一段解释，以及它来自哪里。它会追踪宪法式规则和它们的优先级。也会追踪基准测试用例和它们最新的结果。

那个范围的区分很重要。全局知识会应用在一个 agent 跨越每一个项目。那里放着默认的系统提示，连同核心的参考文档、通用的规则和基础例子。它通常由平台管理员，或者负责维护这个 agent 的人去管理。至于项目专属知识，就只应用在一个项目里面。那里放的是一个品牌自己的风格指南，连同只在那个项目里才说得通的修正、配合它类型和调性的自定义例子，以及项目专属的规则。它由项目所有者或者编辑去管理。在运行的时候，两者会合并，而当它们有冲突的时候，项目知识会胜出。所以如果一条项目规则说所有东西都要竖向，那就会覆盖全局的默认。

这里还有一个很强的概念，就是自动学习。系统不是只坐着等你去教它。它会观察制作历史，自己提出改进建议。如果一个 agent 在多个制作里，三次或以上通不过同一个标准，它就会建议加一条关于那个模式的规则。如果你不停用同一种方式覆盖一个 agent 的输出，它就会提出从你的模式做一条修正出来。如果一个 agent 的作品不停被人用同一种方式批评，它就会建议把这样东西变成一个例子。而当多次否决都带着同样的反馈的时候，它可以自动建立修正条目，并标记为等着你检视。这些会以友好建议的形式出现。比如，这个 agent 为竖向内容用了宽银幕四次，想不想加一条规则？又或者，剪辑师批评了节奏六次，想不想加一个例子？又或者，夜晚场景的匹配分数不停偏低，想不想加一份参考指南？在每一个情况里，你都可以在决定之前接受、驳回，或者看回那些具体例子。

所以总的来说，你有三种模式，它们像一个团队一样合作。配置，你在这里查看和编辑提示、工具、评分标准、关系和模型，而且不用花钱。Playground，你在这里用自己的输入单独测试一个 agent，每次运行成本从零到几块钱都有。以及知识，你在这里添加文档、例子、修正和规则，除了存储之外不用任何钱。拼在一起，这个节奏很简单。你在制作里看到一个问题，你在知识里加一条修正，你在 Playground 测试那个修正，你用一个基准去验证它，而从那一刻起，你的 agent 对之后每一个制作来说都变得更聪明。
