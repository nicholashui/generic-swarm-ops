我哋一齊行一次 VA Agent Swarm 嘅完整介面設計。呢個佈局,就係將一個由超過一百個專門 agent 組成嘅系統,收納喺一個易明嘅工作空間入面。佢涵蓋咗系統 composition diagram 入面每一個操作,亦都帶你由最初嘅一個念頭,一路去到一條製作完成、交付完畢嘅片。而因為呢個平台支援十種唔同嘅工作流程,由 A 排到 J,所以個設計要識得因應每一種而變化。

我哋就由背後嘅理念講起,因為每一個決定都係由幾條核心原則衍生出嚟。

第一條原則係 brief 行先。每一條製作都由一份人手寫嘅 brief 開始,即係有人真正想創作嘅描述。所以個介面就將呢份 brief 擺喺重心位置,輸入佢就係其他一切自然嘅起點。

第二條原則係漸進式披露。成百個 agent 一齊運作,如果一次過全部顯示出嚟就會令人喘唔過氣。所以個設計只會顯示當前階段真正需要嘅嘢,其餘嘅就收埋,等有需要先至出現。

第三條原則係令 agent 圖即時可見。Composition diagram 並唔係一幅靜止嘅圖,佢係即時運行嘅。用家要望一眼就即刻明白每個 agent 而家喺度做緊乜。

第四條原則係由 gate 帶動嘅信心。喺關鍵時刻,一個 gatekeeper agent 會判斷條製作係咪準備好由一個階段過去下一個階段。呢啲轉換喺介面上會以清晰、明確嘅核准時刻出現,等人可以有信心咁簽核。

第五條原則係 critique 透明度。Agent 之間不斷互相評論大家嘅成果,而每一條呢類訊息都可以查睇、可以搜尋,亦都係你可以採取行動嘅對象。

第六條原則係因應製作類型而變。十種工作流程範本決定咗邊啲 agent 會被喚醒、邊啲面板會出現,所以個介面會根據你做緊乜而重新塑造自己。

咁呢一切又係畀邊個用呢?主要有四類用家。創作者想快速開始一條製作、檢視成果、核准各個 gate。監製就睇實預算同進度、處理升級嘅問題、管理團隊。技術操作員調校 prompt、檢查 agent 嘅日誌、管理模型點樣分派。而檢視者或者客戶就睇交付物、留低意見,畀最終核准。

我哋睇吓成個應用程式點樣編排。最頂係 dashboard,即係你嘅大本營,有一個進行中製作嘅格仔陣、一個畀新 brief 用嘅快速開始精靈,同埋一條顯示整體系統健康狀況嘅橫額。由嗰度你會進入 Brief Studio,喺度揀範本,以結構化同自由發揮兩種方式寫 brief,上載參考素材,例如情緒板同劇本,然後確認啟動,將一切交畀 planner agent。

真正嘅主力係 Production Console,每一條製作都有一個。佢入面有 DAG canvas,即係即時運行嘅 composition diagram,顯示各個 agent 節點處於唔同狀態、artifact 由一個 agent 交去另一個嘅流動,以及成果獲核准嘅 gate 檢查點。亦有一個 timeline 檢視,以階段泳道由前期製作一路行到製作、後期同交付,仲有里程碑標記同預算消耗疊層。仲有一個 agent inspector,畀你深入睇任何一個 agent,顯示佢嘅身份同角色、佢當前嘅任務同進度、佢嘅輸入同輸出 artifact、佢發出同收到嘅 critique、佢相對於門檻嘅質素指標,以及佢嘅工具呼叫紀錄。仲有一個 artifact 圖庫,有版本歷史、影片音訊圖像同文字嘅預覽、來源鏈,以及並排比較模式。仲有一個 critique feed,顯示按時間順序排列嘅訊息流,有篩選功能同一個畀人手介入嘅位置。仲有一個 gate 控制面板,擺住待核准嘅排隊、準則清單、核准、拒絕同要求修改嘅操作,以及來源簽核。

喺 console 之外,仲有一個 agent registry,列出所有 agent,可以搜尋同篩選,每個都有詳情卡、依賴關係圖同效能基準。仲有一個記憶同知識區,擺住專案記憶、一份反思嘅情節日誌、一本用嚟構建世界觀嘅系列聖經,以及一個品牌素材庫。仲有一個交付中心,用嚟建立母片包、製作各頻道專屬嘅變體、追蹤品質管控狀況、監察發行,以及檢視發佈後嘅分析。仲有一個設定同管理區,處理模型分派規則、成本同延遲預算、API 金鑰管理、團隊權限,以及合規設定。最後仲有說明同文件,包括一份 agent 詞彙表、一份工作流程範本指南,以及一份 API 參考。

而家講吓 master shell,即係收納住一切嘅外框。橫跨頂部嘅係一條大約六十四像素高嘅橫條,擺住標誌、一個你用 command-K 就觸發到嘅全域搜尋,可以跨 agent、artifact 同 critique 搜尋,仲有通知、用戶選單同設定。左邊係一條窄窄嘅導覽欄,預設只得圖示,當你將滑鼠移上去就會展開顯示標籤。頂部橫條之下係一條情境列,扮演當前製作嘅麵包屑,顯示佢嘅名稱、階段、預算同健康狀況。畫面大部分都係主檢視區,即係當前嘅工作空間。而由底部向上滑出嘅,係一個詳情抽屜,最多可以佔半個畫面,等你可以檢視一個 agent 或者預覽一件 artifact 而唔使離開原本嘅位置。喺最底嘅係一條幼身嘅狀態列,一眼就睇到即時遙測數據,例如有幾多個 agent 運行緊、當前嘅階段、相對於上限已用咗幾多預算,以及估計仲要幾耐。

導覽係分層運作嘅。最高一層,你用側邊導覽欄喺各個應用程式區塊之間移動。喺一個區塊入面,你用一條分頁列切換檢視,喺 DAG canvas、timeline、圖庫同 critique feed 之間揀。再深入啲,詳情會喺抽屜或者彈窗入面打開,例如 agent inspector 或者 gate 核准對話框。而最深嗰層,就係透過右鍵選單或者命令面板嘅情境操作,例如重試一個 agent、比較版本,或者匯出一件 artifact。

介面入面每一個界面都直接對應到 composition diagram 入面嘅操作。Brief 精靈係入口,將用家連去 planner agent。範本選擇器喺十種工作流程類型之間揀選。DAG canvas 呈現由 planner 去 orchestrator 去 router,再去到各 craft agent 嘅完整流程。一個個節點卡顯示每個 agent 嘅狀態。Gate 核准對話框處理由 gatekeeper 同 judge agent 帶動嘅階段轉換。Critique feed 承載住所有 agent 共用嘅訊息匯流排。記憶面板服務 memory agent。Agent inspector 處理深入鑽研。Artifact 圖庫同檢視器顯示同比較五十二個 craft agent 嘅成果。仲有界面負責 timeline、預算追蹤器、router 設定、一個 prompt 實驗室、一個質素儀表板、一個交付封裝器、一個分析面板、一個合規檢查器,以及專門畀創意、研究同優化 meta-agent 用嘅面板。仲有一個通知中心、一個團隊同權限區,以及一個畀長期情節記憶用嘅系列聖經編輯器。

Dashboard 一開始就有一個快速開始區,包括一個開新製作嘅按鈕同一句邀請你描述想創作乜嘢嘅提示,旁邊係一行由 A 到 J 嘅範本卡,例如爆紅鈎子、UGC 廣告、解說片、生日片、短片、企業片同音樂錄影帶。下面係一個進行中製作嘅格仔陣,每張卡顯示專案名稱、佢嘅工作流程類型、當前階段、一條進度條、完成百分比,以及到目前為止用咗幾多錢。最底係一條系統健康狀況帶,顯示有幾多 agent 在線、有幾多 gate 待處理,以及有幾多警示生效中。撳一張範本卡會打開預先載入咗嗰個工作流程嘅 Brief Studio。撳一張製作卡會打開佢嘅 console。而由零開始就會打開一份空白嘅 brief。

Brief Studio 會引導你完成幾個步驟。首先你揀一個範本,studio 會預覽嗰個範本會啟動邊啲 agent。然後你填寫 brief 詳情:一個標題、一段兩三句自由發揮嘅願景陳述、類型、用一條由十五秒伸展到兩個鐘嘅滑桿揀時長、長寬比、以一組標籤表示嘅基調、目標觀眾、一個預算上限,以及一個死線。跟住你加參考素材,拖入劇本、情緒圖像、參考影片同音訊,或者由品牌庫拉素材出嚟,你亦可以貼上風格參考連結或者揀配音同演員偏好。之後你設定限制同合規,有啲開關處理例如來源簽署、無障礙標準、AI 同意核實、資料處理規則,以及你針對邊啲平台。最後你檢視同啟動,studio 會顯示一個計劃預覽,估計 planner 會建立幾多個階段、會涉及幾多 agent、預期成本同預期時間。由嗰度你可以儲存草稿,或者啟動製作。

每個範本都有自己嘅預設。爆紅鈎子範本預設係短身直度影片,針對 TikTok 同 Reels,並啟動鈎子、趨勢、社交同留存 agent。UGC 廣告範本聚焦喺成效目標,引入品牌、文案同 A/B 測試 agent。動畫解說片偏向教育,有動畫、動態圖像同旁白。個人化生日範本加入個人化變數、範本、虛擬人同配音。AI 短片解鎖完整嘅創意團隊,加上攝影、剪接同聲音。企業培訓範本加入學習管理封裝同評估。音樂錄影帶帶嚟節拍同步同編舞。虛擬人範本加入聲音複製同對嘴。紀錄片範本帶嚟檔案研究同事實查核。而劇情長片範本就啟動整個 agent 陣容,每一個 gate 都會運作。

Console 嘅核心係 DAG canvas。喺呢度,即時運行嘅 composition diagram 會變成一個你可以探索嘅互動圖。你會見到頂部嘅 brief,向下流去 planner agent,然後係 orchestrator,佢分支去 router 同 memory agent,再由嗰度去到各 craft agent,例如導演、攝影指導、剪接師、作曲家同視覺效果總監。佢哋下面坐住 gatekeeper,佢喺度等核准,以及 judge,佢忙緊評分。每個節點都帶有清晰嘅狀態。一個剔代表完成,顯示綠色。一個實心點代表運行中,會閃動藍色。一個空心點代表閒置或者排緊隊,顯示灰色。一個警告符號代表受阻或者等緊核准,顯示琥珀色。而一個交叉代表失敗,顯示紅色。你可以撳一個節點,喺抽屜入面打開佢嘅 inspector;雙撳就會有全螢幕檢視;撳一條邊就睇到正喺傳遞嘅 artifact;撳 gatekeeper 就打開佢嘅核准對話框;或者右撳就有情境選單,有重試、跳過、檢查同睇 critique 等選項。你拖曳就可以平移,捲動就可以縮放,撳住 control 就可以一次過揀幾個節點。仲有篩選,可以顯示所有 agent、只顯示運行緊嘅、只顯示受阻嘅,或者只顯示已完成嘅,加埋圖層開關,分別控制 orchestration、craft、創意、研究同優化 meta-agent,以及 critique 圖層。

Timeline 檢視沿住一條橫向時間軸講返同一個故事。每個階段都有自己嘅泳道:前期製作、製作、後期製作同交付,有填充嘅長條顯示進度,亦有標籤標示每個階段入面嘅步驟。Gate 標記坐喺底部,顯示邊啲 gate 已經通過、邊啲喺檢視中、邊啲仲未處理。而一條預算消耗線就描繪住開支點樣隨住製作向前推進而上升。

Artifact 圖庫將所有 agent 製作出嚟嘅嘢以一個格子或者清單卡片呈現。每張卡顯示一個縮圖,可能係影片畫格、音訊波形、圖像,或者文件圖示,連同一個版本徽章同佢嘅狀態、製作佢嘅 agent、一個關鍵質素指標,以及一個確認佢已經簽署嘅來源徽章。撳一張卡會喺抽屜入面打開 artifact 檢視器,喺度你可以並排比較版本、瀏覽完整版本歷史,以及追蹤完整嘅來源鏈。

Critique feed 係 agent 之間一場活生生嘅對話。每一條紀錄顯示時間、邊個對緊邊個講嘢,以及一個嚴重程度,由資訊性到警告到嚴重不等。舉個例,一個剪接 agent 可能會話畀導演聽某場戲嘅節奏拖得有少少長,建議剪走一啲片段,並附上一條節奏曲線做證據。一個質素 agent 可能會標示某幾個畫格出現手部瑕疵,建議用新嘅 seed 重新生成。一個合規 agent 可能會警告所有人話一個聲音複製嘅同意快將過期,並喺續期之前封鎖交付。而 judge 可能會宣佈一場創意辯論已經解決,解釋邊一方贏,以及贏咗幾多分。每條訊息都附帶操作,例如接受、拒絕、討論、自動修復,或者升級畀人處理。而喺底部坐住一個人手介入嘅位置,你可以喺度向任何 agent 打你自己嘅 critique 或者指示、直接提及佢哋,並設定優先級。

當一個階段準備好推進,gate 核准對話框就會出現。佢會講明係邊個 gate、係邊個轉換,顯示 gatekeeper 嘅評估同 judge 嘅數字分數,並列出一份準則清單,每一項都標示為通過或者需要注意,例如劇本覆蓋、對白獨特性、分鏡完整度、預算喺上限之內、風格一致性,以及一個合規預檢。佢列出待檢視嘅 artifact、提供一個留言框,並畀你三個核心操作:核准、拒絕,或者要求修改。佢亦都確認來源簽署,將呢次簽核歸於你。

Agent inspector 就係你深入認識一個 agent 嘅地方。佢顯示佢嘅身份,包括佢嘅類別同佢遵循嘅推理模式、佢由幾多個 agent 接收工作,以及佢對幾多個 agent 作出評論。佢顯示佢當前嘅任務同狀態,包括佢做緊第幾次自我修煉。佢以長條形式顯示質素指標,對照住門檻,例如文字對圖像嘅吻合度、節拍覆蓋同節奏吻合。佢列出輸入同輸出 artifact。佢記錄工具呼叫,例如一個向影片生成模型發出嘅請求,或者一次記憶調取。而佢亦顯示 critique 匯流排,收到同發出嘅都有,等你睇到佢同其他 agent 之間嘅一來一往。由呢度你可以重試任務、跳過佢、發出 critique,或者睇完整歷史。

Agent registry 係系統入面每一個 agent 嘅目錄,分門別類,例如線上創作層、攝影、剪接、聲音、表演、發行、教育、AI 專家,以及各種 meta-agent 組別,分別負責 orchestration、創意、研究同優化。每一行顯示 agent 嘅編號、佢嘅名稱、佢嘅推理模式、佢用嘅工具,以及佢當前嘅狀態。撳任何一行就會打開一張顯示佢完整能力嘅詳情卡。

記憶同知識面板擺住系統嘅長期理解。專案記憶可以用語意同精確兩種方式搜尋,並顯示最近嘅條目,例如關於某場戲基調嘅筆記、某個角色嘅服裝規則、一次預算修訂,或者一個鎖定咗嘅風格 seed,連同邊啲 agent 一直喺度存取佢哋、存取咗幾多次。仲有畀情節日誌、系列聖經、品牌庫,以及一個世界觀資料庫嘅區段。

交付中心管理將完成嘅作品送出去呢件事。一個頻道矩陣列出每個出口,例如 YouTube、TikTok、Meta、廣播、戲院同檔案,並就每一個顯示格式、品質管控狀況、字幕、無障礙級別、來源,以及一個整體已準備好或者待處理嘅狀態。一份品質管控摘要將檢查分成三個級別:技術層面,例如響度、色彩同解析度;創意層面,例如節奏、節拍同步同風格;以及合規。由呢度你可以封裝所有已準備好嘅嘢、修復待處理嘅問題,或者打開完整嘅品質報告。

設定入面嘅 router 設定,就係你決定邊個模型處理邊啲任務嘅地方。一個路由規則表將每種任務類型,例如影片生成、聲音合成、虛擬人渲染、圖像生成、創意語言模型、judge 模型同音樂生成,同一個主模型、一個後備,以及每項任務嘅最高成本配對埋一齊。下面係成本防護:一個全域預算上限、一個警示門檻、一個質素會自動降級嘅點,以及一個需要人手覆寫嘅硬性停止點。

Prompt 實驗室係用嚟製作同精煉生成 prompt 嘅工作空間。你揀一個 agent、一個模型同一個鏡頭,然後寫個 prompt 描述你想要嘅場景,並設定參數,例如 seed、引導、步數同負面詞。你可以生成、跑一次優化、做 A/B 測試,或者逐個 seed 行一遍。一個優化歷史表追蹤每一次迭代、佢嘅分數、改咗乜嘢嘅摘要,以及佢有冇被接受,等你睇到個 prompt 隨時間點樣改善。而一行生成出嚟嘅成果,就畀你比較各次迭代,並將最好嗰個送畀導演核准。

質素儀表板將所有評估分數匯總。佢以長條形式顯示整體指標,對照住佢哋嘅門檻,涵蓋例如影片基準分數、文字吻合度、動態質素、美學、音訊清晰度同響度。佢將結果逐個鏡頭分拆,標示手部、面部辨識、時間一致性同風格。並浮現回退警示,精準指出邊度個分數跌咗,有選項可以全部自動修復、人手檢視,或者撤銷啲輕微嘅。

而家追蹤一次完整嘅製作開始流程,即係由念頭去到完成片嘅旅程。佢由 dashboard 開始,喺度你可以撳一個範本、由零開始開一條新製作,或者打入全域搜尋,等系統建議一個範本。跟住係 Brief Studio,喺度你確認範本、填寫 brief、上載參考素材、設定限制,並檢視計劃預覽。然後你啟動:brief 去 planner agent,佢將佢拆成一個分階段嘅圖;planner 交畀 orchestrator,佢初始化執行;orchestrator 呼叫 router 去分派模型同 agent;memory agent 用 brief 同參考素材播種;而個介面就滑入 production console。

由嗰度,條製作會經歷佢嘅各個階段,大部分自動進行,但喺各個 gate 都有人喺循環之中。喺前期製作,agent 寫劇本、畫分鏡、開發外觀、揀配音,並勾勒初步嘅音樂主題,而創意同研究 meta-agent 就餵入諗法同情境。Gatekeeper 檢查第一級準則,一個 gate 對話框出現,你就核准。喺製作階段,導演發出鏡頭意圖,prompt 工程師製作 prompt,router 將佢哋送去影片模型,攝影指導核實構圖,而質素 agent 就逐格檢查,優化 agent 就一路行一路調校。Critique 訊息流動,artifact 喺圖庫出現,你可以透過 critique feed 介入。又一個 gate,又一次核准。喺後期製作,剪接師組裝剪輯,調色師套用色調,聲音團隊鋪入音訊並做混音,視覺效果總監做合成,而一個無障礙 agent 就加入字幕同口述影像。Judge 對照一個評分準則評分,gatekeeper 檢查最後嘅準則,你又再核准。然後就到交付,中心顯示頻道矩陣,發行商就每個出口封裝,合規畀一個最終法律簽核,來源喺所有嘢上面簽署,你就核准發行。而可選地,喺發佈之後,一個分析師收集成效數據,優化 agent 研究留存同廣告支出回報率,分析面板顯示結果,而啲學到嘅嘢就回饋入記憶,畀未來嘅製作用。

唔同範本會以一啲顯著嘅方式改變體驗。爆紅鈎子將一切壓縮喺短短幾分鐘之內,將趨勢情報擺喺最前最中,並預覽留存曲線。UGC 廣告突出成效目標,並顯示廣告支出回報嘅預測。動畫解說片加入一個學習目標編輯器同一個評估建立器。個人化生日加入一個變數範本編輯器同一個批次渲染排隊。短片顯示完整嘅圖,五十二個 craft agent 全部啟動。企業培訓加入學習管理封裝同學員模擬。音樂錄影帶加入一個節拍格同波形疊層。虛擬人範本加入一個虛擬人設計工作室同一個同意流程。紀錄片加入一個檔案研究面板同來源評級追蹤。而劇情長片就加入集數同幕次導覽、系列聖經、完整陣容,甚至電影節策略。

為咗確保系統入面冇嘢被遺漏,composition diagram 入面每一個節點同每一條邊都對應到介面入面具體嘅界面。Brief 對應到精靈同範本選擇器。Planner 出現喺 canvas、timeline 同 inspector。Orchestrator 出現喺 canvas、timeline、通知中心同預算追蹤器。Router 出現喺 canvas、router 設定同優化面板。五十二個 craft agent 出現喺 canvas、佢哋嘅節點卡、inspector、圖庫、檢視器、prompt 實驗室同質素儀表板。Critique 訊息流經 feed,並進入每個 inspector 嘅 critique 分頁。Judge 同 gatekeeper 出現喺 canvas 同 gate 對話框,gatekeeper 仲連住合規。Memory agent 對應到記憶面板同系列聖經。而每一族 meta-agent,創意、研究同優化,都有自己專屬嘅面板,有啲控制例如腦力激盪卡、節拍表視覺化、風格鎖、情緒板組合器、趨勢時間線、競爭對手格、來源評級徽章同留存預測器。

講幾句關於反應式同無障礙。個佈局會跨斷點適應:喺大型桌面所有嘢一次過可見;喺標準桌面導覽欄收細成圖示,抽屜疊喺 canvas 上面;喺平板就變成單一窗格加分頁;而喺手機就 dashboard 同 brief studio 維持可用,而 production console 就變成唯讀,用嚟監察。無障礙係認真對待嘅,目標係達到最新網頁內容指引嘅雙 A 級別。文字符合嚴格嘅對比度,狀態除咗用顏色亦用形狀傳達,成樣嘢都可以用鍵盤導覽,有可見嘅焦點環同命令面板,螢幕閱讀器會得到適當嘅地標同即時區域,會宣讀 agent 狀態嘅變化,亦有一個減少動態嘅模式,將動畫變成靜態轉換。各個節點狀態刻意用唔同嘅形狀,所以佢哋從來唔係只靠顏色,呢樣對色盲用家嚟講仍然清晰可讀。深色同淺色主題都支援,而 canvas 喺兩種模式都用一個柔和嘅背景,等節點保持可見。

呢個設計建基於一個可重用組件嘅程式庫。有 agent 節點卡、artifact 卡、critique 訊息、gate 檢查點、指標長條、timeline 泳道、brief 欄位、抽屜面板、一個命令面板、通知徽章、來源徽章同預算量規,每個都有自己一套變體。而呢啲會組合成複合模式:agent inspector、gate 對話框、prompt 編輯器、頻道行同 timeline 階段,每一個都係由啲較細嘅部件砌成。

互動模式將一切串連埋一齊。命令面板,用 command-K 觸發,係一個喺任何地方都去到嘅全域搜尋同操作啟動器,顯示最近嘅項目、常用操作,以及一個即時篩選嘅 agent 清單。人喺循環之中嘅時刻處理得好優雅:當去到一個檢查點,gate 對話框就出現;當一個 agent 需要輸入,critique feed 就會突顯;當踩到預算門檻,一條橫額就警告;當有合規封鎖,一個彈窗就浮現;而質素儀表板就標示回退。即時更新令一切保持最新:agent 狀態每隔幾秒平順過渡,critique 訊息透過 websocket 滑入,artifact 完成時動畫式滑入圖庫,預算量規逐項任務更新,而質素長條就隨每次評估移動。而批量操作畀你一次過重試所有失敗嘅 agent、核准所有符合門檻嘅待處理 gate、匯出選定或者全部 artifact、比較任何兩個版本,或者清走過時嘅記憶。

仲有一個通知同警示系統,有清晰嘅優先級。嚴重警示,例如一個合規封鎖或者預算超支,會佔據整個畫面,連同一下提示音,而且唔採取行動就唔可以撤銷。高優先警示,例如一個 gate 準備好核准,會以一個帶徽章嘅吐司顯示,並要求喺幾分鐘之內採取行動。中等警示,例如一項完成嘅任務或者一條新 critique,只係令一個徽章遞增。而低優先警示,例如一個優化建議,就只顯示一個徽章。通知中心將佢哋全部收集埋,將未讀同已讀分開。

喺底層,個介面為每一條製作維持一個結構化嘅狀態,透過 websocket 同後端保持同步。一個製作物件擺住佢嘅身份、佢嘅範本、佢當前嘅階段、brief、agent 圖、預算、各個 gate、artifact、critique、記憶、交付狀態,以及可選嘅分析。個圖本身追蹤節點同邊、當前階段同一個完成百分比。每個 agent 節點帶有佢嘅身份、佢嘅狀態、佢當前嘅任務、佢嘅指標、佢發出同收到嘅 critique、佢嘅工具呼叫,以及佢嘅 artifact。Gate 追蹤佢哋嘅準則、judge 嘅分數、佢哋嘅狀態、邊個核准佢哋,以及佢哋有冇被簽署。Artifact 追蹤佢哋嘅版本、類型、製作嘅 agent、質素分數、來源狀態同父素材。而 critique 訊息追蹤發送者、接收者、嚴重程度、內容、任何附件、狀態,以及任何人手回應。

至於技術,個設計建議用一個現代嘅網頁技術棧:用 React 同 Next.js 做框架,將畀 dashboard 用嘅伺服器渲染同畀即時 console 用嘅客戶端互動融合埋一齊;以 websocket 同步嘅輕量狀態管理;一個專為渲染 DAG 而設嘅節點圖程式庫;畀 timeline 用嘅自訂視覺化;畀影片同音訊預覽用嘅強勁播放器;畀一切即時更新用嘅 websocket;一個無障礙、以功能為先嘅樣式手法;畀指標同分析用嘅製圖程式庫;一個經得起考驗嘅命令面板組件;一套乾淨嘅圖示;以及喺 prompt 實驗室入面一個完整嘅程式碼編輯器,用嚟編輯 prompt 並有語法高亮。

最後,個設計指向一啲配套嘅線框檔案:一個畀完整應用程式外殼,一個將 composition diagram 對應落介面界面。合埋一齊嚟睇,呢個設計將一個極之複雜嘅 agent 群體,變成一個人類真係可以理解、引導同信任嘅工作空間,由最初嗰份 brief,一路去到完成、交付嘅電影。
