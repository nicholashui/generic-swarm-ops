嗰個範圍嘅區分好重要。全域知識會套用喺一個 agent 跨越每一個項目。嗰度住住預設嘅系統提示，連同核心嘅參考文件、通用嘅規則同埋基礎例子。佢通常由平台管理員，或者負責維護個 agent 嘅人去管理。至於項目專屬知識，就淨係套用喺一個項目入面。嗰度放嘅係一個品牌自己嘅風格指南，連同淨係喺嗰個項目先講得通嘅修正、配合佢類型同調子嘅自訂例子，同埋項目專屬嘅規則。佢由項目擁有者或者編輯去管理。喺執行嗰陣，兩者會合併，而當佢哋有衝突嗰陣，項目知識會勝出。所以如果一條項目規則話所有嘢都要直向，咁就會覆寫全域嘅預設。

呢度仲有一個好強嘅概念，就係自動學習。系統唔係淨係坐喺度等你去教佢。佢會觀察製作歷史，自己提出改進建議。如果一個 agent 喺多個製作度，三次或以上過唔到同一個準則，佢就會建議加一條關於嗰個模式嘅規則。如果你不停用同一種方式覆寫一個 agent 嘅輸出，佢就會提出由你嘅模式整一條修正出嚟。如果一個 agent 嘅作品不停被人用同一種方式批評，佢就會建議將呢樣嘢變成一個例子。而當多次否決都帶住同樣嘅回饋嗰陣，佢可以自動建立修正條目，並標記為等緊你檢視。呢啲會以友善建議嘅形式出現。譬如，呢個 agent 為直向內容用咗闊銀幕四次，想唔想加一條規則？又或者，剪接師批評咗節奏六次，想唔想加一個例子？又或者，夜晚場景嘅配合分數不停偏低，想唔想加一份參考指南？喺每一個情況，你都可以喺決定之前接受、駁回，或者睇返啲具體例子。

所以總括嚟講，你有三種模式，佢哋係一隊咁合作。配置，你喺度查看同編輯提示、工具、評分準則、關係同埋模型，而且唔使錢。Playground，你喺度用自己嘅輸入單獨測試一個 agent，每次執行成本由零到幾蚊都有。同埋知識，你喺度加文件、例子、修正同埋規則，除咗儲存之外唔使任何錢。砌埋一齊，個節奏好簡單。你喺製作度見到一個問題，你喺知識度加一條修正，你喺 Playground 測試嗰個修正，你用一個基準去驗證佢，而由嗰一刻起，你個 agent 對之後每一個製作嚟講都變得更叻。
