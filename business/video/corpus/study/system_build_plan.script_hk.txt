第二個合約係評論訊息匯流排架構。呢個就係任何 agent 對任何其他 agent 嘅作品作出評論嘅方式。每一條評論帶住一個 id、佢來自邊個 agent、佢針對邊個 agent、一個對相關產物嘅參照、一個嚴重程度、一個類別、支持嘅證據、一個建議行動、一個可選嘅評分準則參照、一個截止階段、一個可選嘅分數，同一個時間戳。嚴重程度由阻塞級（blocker），向下經過重大（major）、輕微（minor），到最細嗰個小瑕疵（nit）。類別涵蓋節奏、連貫性、準確性、合規、無障礙、品牌、工藝同美學。驗收規則好嚴格而且經過徹底測試。一個阻塞級會令到嗰個工作流程節點停低，直到解決為止。一個重大會喺接收嘅 agent 上面觸發一個自我精煉循環，上限三次迭代，之後就升級交俾一個裁判。輕微同小瑕疵嘅評論會記錄入記憶，並且匯總做下一個訓練週期嘅獎勵訊號。兩個 agent 之間嘅爭議會被路由去一個裁判度做一場有結構嘅辯論，而合規評論永遠當作阻塞級去處理。

第三個合約係共用嘅 agent 基底類別。嗰一百一十四個 agent 每一個都係呢一個類別嘅實例。佢嘅配置會命名個 agent、佢嘅類別、佢嘅系統 prompt、佢嘅模型政策（連同偏好嘅模型同預算）、佢被允許調用嘅工具、佢嘅評分準則、佢嘅可量度自我品質指標、佢嘅評論關係，同佢嘅精煉上限。基底類別本身定義咗生命週期。佢起草作品、對照住評分準則去評論自己嘅草稿，再修訂，跟住 self refine 嘅模式。失敗嗰陣，佢會儲存口頭回饋再重試，跟住 reflexion 嘅模式。基底類別亦都接駁咗嗰個有計量功能嘅語言模型 gateway、檢索客戶端、記憶儲存、event bus、來源證明簽署同追蹤。冇任何子類別會重新實作呢啲嘢嘅任何一部分。Agent 之間嘅唯一分別，就只係佢哋嘅配置。

第四個合約係嗰套供應商介面，全部都可以模擬。第五個係品質網（quality mesh）本身，連同三個評估層，再加一個六道交付檢查。第一層用確定性驗證器去檢查規格合規性，而且必須完全通過。第二層係由一個扮演裁判嘅語言模型，用嗰個角色嘅憲法去畀出嘅評分準則分數，而且必須達到至少一百分之中嘅八十五。第三層係一個偏好比較，對住一個人類參照做兩兩比較，再加上一個至少二百個人物角色嘅模擬觀眾，同至少二十個人機協作樣本，而且必須達到對等或者超越一個門檻。嗰六道交付檢查涵蓋規格、產物完整性、音訊同步、連貫性、感知品質同出街準備，而六道全部都必須通過。嗰啲確定性檢查係最先建造嘅，因為佢哋完全可以測試。嗰啲基於裁判嘅檢查係用凍結咗、釘實咗嘅裁判同黃金測試集去建造，等到啲分數保持穩定。計劃好堅決，就係一個裁判模型永遠都唔應該被允許浮動而冇釘實。

合約 package 亦都會生成前端型別，等到網頁主控台同後端保持同步，並且定義 event bus 嘅主題。每一個製作都有俾 agent 事件、評論、關卡同產物用嘅主題，再加一條系統範圍嘅警示頻道。每一個事件都必須係其中一個有型別嘅事件模型，好似一個 agent 狀態變更、一個產物已建立、一條評論訊息、一個關卡變成準備好或者已解決、一個預算更新、一個指標更新、一個記憶條目、一個工具調用、一個製作階段變更，或者一個錯誤。呢啲有型別嘅形狀，係匯流排上面唯一被允許嘅。

合約就位之後，計劃就鋪陳出一個分階段嘅路線圖，有十三個里程碑，由零編號去到十二。排序原則係由基礎去到智能、去到生產、再去到強化，但係嗰個垂直切片會喺好早期就打穿過去，等到架構喺擴展去廣度之前已經被證明。每一個里程碑都會指明佢嘅目標、佢嘅依賴、要建造嘅檔案、編碼 agent 嘅工作流程、測試，同一個硬性嘅驗收關卡。喺當前嘅關卡變成綠色同記錄低之前，你唔會開始下一個里程碑。

里程碑零係引導啟動。目標係一個可以編譯、綠色、工具齊全但係空白嘅 monorepo，連同編碼 agent 嘅所有配置都就位。暫時冇任何嘢做到啲乜，但係驗證關卡通過，而且啟動嗰個容器堆疊會帶起每一個後端服務。呢度就係建立鷹架、命令介面、容器定義、依賴設置、持續整合管線，同所有編碼 agent 配置嘅地方。個關卡就係：由一個乾淨嘅 clone 開始，引導啟動、帶起服務同驗證全部都通過綠色。

里程碑一係要建立基礎知識服務，亦即係每一個 agent 都會調用嘅 agentic 檢索系統。佢包含一條攝取管道，會將文件切塊、做 embedding，再索引去一個 vector store 同埋一個 graph 層。佢提供混合式檢索，結合 vector、graph 同關鍵字搜尋並加上重新排序，仲可以規劃多步驟嘅檢索。佢嘅 client 提供檢索、多跳複合查詢、攝取，同埋附帶來源評級嘅引用。知識會分隔成唔同嘅 namespace，咁一個專案嘅世界就唔會洩漏去另一個專案嗰度。呢個關卡要求喺一個 golden set 上面達到精確度目標，並且返回帶評級嘅引用。

里程碑二係最大嘅平台里程碑，亦即係編排運行環境，係成個系統嘅跳動心臟。佢會建立帶型別嘅 event bus，具備至少一次嘅交付保證同埋冪等性；亦會建立可觀測性層，喺嗰度每一個節點嘅運行都係一條 trace span；仲有編排服務，結合 graph 運行環境同埋人在迴路嘅中斷點，以及一個持久化嘅 workflow engine，喺嗰度每一個 agent 任務都係一個可重試嘅 activity。佢亦會建立平台 agent 嘅骨架版本，包括 orchestrator、planner、router、judge、gatekeeper 同 memory agent，再加上 memory store 同埋資產骨幹，後者具備不可變嘅 id、寫時複製嘅版本同埋來源簽署。呢個係要「諗到爆」嘅里程碑。個計劃明確要求將 graph framework 同 workflow engine 之間嘅邊界，記錄成一個有案可查嘅決策。佢嘅驗收關卡，亦即係所謂嘅 walking skeleton，係一個寫死嘅三節點 graph，可以喺真實基建上由頭行到尾，捱得住中途殺死一個 worker，發出正確嘅型別事件，並且為佢嘅產物簽署。

里程碑三係要建立語言模型 gateway、router 同埋成本儀表板，咁由第一日開始，每一個 token 都會被計量同埋路由。個 gateway 包住一個統一嘅 client，提供 provider 抽象、自動重試同埋退路、每次調用嘅 token 同成本計量並發送去 bus、將版本標記寫入來源紀錄、回應快取，仲有一個凍結嘅 judge 模式。Router agent 會按成本、品質同延遲揀選最好嘅 agent 同模型，而成本儀表板就會按每個製作、每個 agent、每個 provider 匯總開支，並設有警報門檻。預算護欄會喺一個製作超出佢嘅額度時，強制硬性停止同埋升級處理。呢個關卡要求任何 agent 調用都被計量、被路由、被版本標記、喺儀表板可見，並且喺預算違反時觸發一次真正嘅停止。

里程碑四係要建立 research agent 同埋 coding agent 嘅框架。Research agent 會做查詢規劃、多來源檢索、綜合，並且產出一份帶來源評級同引用嘅檔案，再寫入知識 namespace 入面。有趣嘅係，個計劃指出 coding agent 嘅規格，其實正正描述緊 coding agent 喺呢次建構過程中扮演緊嘅角色，所以佢嘅慣例會被抽取入專案記憶，再由 code reviewer 強制執行。呢個關卡係要產出並儲存一份帶評級同引用嘅檔案，同時 coding 慣例獲得強制執行。

里程碑五係要建立智能層，亦即係每一個製作 agent 都會用到嘅共享推理服務。當中有深度意圖分析，會將一份簡報解析成結構化嘅意圖、目標、受眾、隱藏議程同埋約束，佢係每一個製作嘅入口點。亦有通用創意 agent，係一個創意構思引擎。仲有一個流程優化 agent，會根據遙測數據改善工作流程。亦有一個策略目標達成框架、一套複雜問題解決方法論，以及一個 aesthetics agent，會提供感知評分、新穎性訊號同埋美學評論。Creative agent 同 aesthetics agent 會形成一個生成同埋評估嘅迴路。呢個關卡要求全部六個推理服務都通過佢哋嘅 golden 行為測試，並且要展示出嗰個迴路。

里程碑六係關鍵嗰個，亦即係 agent factory 加上垂直切片。目標係喺建立餘下嗰一百零九個 agent 之前，先用最平嘅真實工作流程去驗證成個架構。個 factory 會將一份 agent 配置變成一個可運行嘅 agent，驗證佢嘅引用並且註冊佢。然後就會經個 factory 建立啱啱好夠用嘅工藝 agent，去由頭行一個工作流程到尾，亦即係病毒式 hook 短片，喺真實基建上用 mock 生成器運行。成條完整路徑會由簡報開始，經過意圖分析，經過一個會建構工作流程 graph 嘅 planner，經過喺 mock 上執行嘅各個 agent，產物會經由交接契約流動，critique bus 處於啟動狀態，品質網把關，產出一份已簽署嘅交付物，並且喺 bus 上有事件。呢個係要「超級深思」嘅里程碑，因為個 factory 最終一定要產出全部一百一十四個 agent，所以佢嘅配置 schema 而家就要做到完整。佢嘅驗收關卡係關鍵性嘅。由頭行到尾運行成個工作流程，一定要由一份簡報產出一份已簽署嘅交付物，每一份交接契約都填好，每一個關卡都有強制執行，所有事件都發出，有完整嘅來源紀錄，並且喺預算之內，全部用 mock provider。個計劃講得好強硬。呢個關卡係用嚟驗證成個平台，而喺佢未夠穩陣之前，你係唔好行去鋪闊度嗰一步。

里程碑七係要建立製作工藝 agent，亦即頭五十二個，做法係用 factory 產出嘅配置、評分準則同埋提示。對每一個 agent，都會用一套可重複嘅劇本去產出佢嘅配置、佢嘅版本化提示、佢嘅評分準則、佢嘅可量度自我品質指標、佢嘅工具許可清單，同埋佢嘅 critique 關係。佢哋會按類別逐個逐個咁建立，共用評分準則嘅鷹架，而且喺類別之間會清空上下文。呢個關卡要求全部五十二個都已註冊，並且喺佢哋嘅規格測試、評分準則測試同 critique 測試上都係綠燈，再加上至少三個額外嘅工作流程原型喺 mock 上由頭行到尾。

里程碑八會將平台 agent 嘅骨架升級成完整實作，並且加入 meta agent，即係第五十三到第八十個。嗰啲涵蓋具備完整爭議解決嘅編排 agent、主要係委派去 creative 同 aesthetics 服務嘅創意 meta agent、建基於共享 research 核心嘅 research meta agent，以及涵蓋提示、成本、延遲、留存、廣告支出回報同埋無障礙嘅優化 agent，再加一個評估框架同埋一隊安全紅隊。完整嘅品質網會喺呢度完成，包括模擬受眾層同埋全部六個交付驗證器，由 gatekeeper 強制執行零洩漏缺陷。個計劃建議喺呢個里程碑早段就建立評估框架同埋安全紅隊，咁佢哋就可以持續測試其他所有嘢。呢個關卡要求全部八十個 agent 上線，完整嘅品質框架喺每一條發佈路徑上都有強制執行，並且紅隊同框架每晚運行。

里程碑九係要建立工作流程支援 agent，即係第八十一到第一百一十四個，以及交付織體。嗰啲包括資產管理、版本控制同埋渲染派發，配一個會跨圖形處理器分批處理同埋自動擴展嘅 render farm。亦包括品質關卡、交付打包同埋合規，會打包成戲院、串流、廣播、存檔、預告片同埋社交格式，附帶各個渠道專屬嘅規格、字幕、元數據、版權管理同埋來源負載。仲包括分析、回饋同埋重新訓練 agent，會將啲細小嘅 critique 匯總成一個獎勵訊號。交付織體會運行一條分支管道，咁戲院、串流、廣播同存檔，再加上行銷衍生品，就會並行產出，並且逐個渠道做驗證。呢個關卡要求一個製作要產出全部四條交付分支再加行銷衍生品，每一個都有效，並且發佈後嘅遙測數據會回流去一張重新訓練嘅工單。

里程碑十係要建立用戶介面、控制台、api gateway 同埋 websocket 層，亦即係人類操作員嘅介面。值得留意嘅係，呢個喺里程碑三之後就可以並行開始，喺一個獨立嘅工作樹入面，先對住一個假嘅事件發射器開發，之後先整合。個 gateway 會完全按照契約暴露 rest 端點同埋 websocket gateway，具備身份驗證、基於角色嘅存取控制、速率限制、驗證，以及喺關卡批准時嘅簽署。佢會訂閱 event bus、按製作過濾，再經 websocket 扇出去。網頁控制台會提供一個簡報工作室、一個顯示節點狀態嘅即時 graph 畫布、一個產物畫廊、一個 critique 動態、一個關卡批准對話框、一個預算追蹤器、一個品質儀表板、一個 agent 檢視器、一個記憶面板，同埋一個交付中心，全部都由型別事件同埋生成嘅型別驅動。呢個關卡係要一個人類可以完全透過瀏覽器，去為第一個工作流程啟動、即時監控、做 critique、批准關卡，並且下載交付物，而且即時更新要好快，亦冇任何 agent 直接調用介面。

里程碑十一係增強層，加入個人化同埋以音訊為先嘅變體。當中有創作者嘅心理剖析、預測受眾偏好嘅心理推薦、一個可以喺嚴格私隱同埋同意審計之下更換姓名、面孔同聲音嘅個人化引擎，仲有一個用嚟做以音訊為先工作流程嘅 podcast agent。呢個關卡要求喺同意關卡之下產出個人化同埋按受眾調節嘅變體，並且要 podcast 原型由頭行到尾。

里程碑十二係加固、規模、安全同埋發佈，而呢個里程碑就係逐個主題咁執行嗰張一百點清單。佢涵蓋負載測試同埋自動擴展、安全加固（包括提示注入防禦同埋一次完整嘅紅隊掃蕩）、透過混沌測試同埋災難復原演練達致嘅可靠性、每一件可發佈產物都有來源紀錄嘅合規、經過驗證嘅成本儀表板、更新嘅文件同埋操作手冊，以及一個由內部到有限再到普遍可用嘅分階段發佈。最後嗰個關卡要求全部一百項檢查都通過，要有一次完整劇情片嘅演練喺 mock 上操練全部一百一十四個 agent，並且要喺真實 provider 上做一次即時冒煙測試，喺預算之內成功。

個計劃停低嚟解釋點解垂直切片要擺喺呢個位置。喺驗證一個工作流程由頭到尾之前就去建立一百一十四個 agent，會係分散式系統嗰個典型錯誤，亦即係喺大部分程式碼已經假設咗某個架構之後，先發現嗰個架構有缺陷。所以個計劃刻意先建立平台，將一條薄切片由頭打通到尾，然後喺架構已經身經百戰嘅情況下，再經 factory 去鋪闊度，之後先加介面同埋增強，最後先加固。如果切片揭示到問題，你係喺得九個 agent 進行中嘅時候去修，而唔係一百一十四個。佢稱呢個為成個計劃入面最重要嗰一個排序決策。

核心位置有一個可重複嘅模式。十個工作流程原型每一個都變成一個 graph。佢哋共用一個共通骨架，即係 greenlight、前期製作、製作、後期、審查同發佈、分發，同埋發佈後，而佢哋唯一嘅分別就只係邊個 agent 領導每一個階段，以及邊啲 critic 為交接把關。一個工作流程係喺佢可以喺 mock provider 上由頭行到尾、每一個階段關卡都強制執行佢嗰套 critic、並且最終產物帶住一條完整嘅來源紀錄鏈之時，先算係完成。

Agent 實作劇本被詳細寫明，係對全部一百一十四個 agent 都會運行嘅確切食譜，冇任何一個 agent 係喺劇本以外用人手砌出嚟。啲步驟係：透過 spec reader 讀取嗰個 agent 嘅規格、將佢嘅自我品質準則映射成帶門檻嘅具體指標、引用規格入面點名嗰個工藝權威去撰寫佢嘅評分準則或者章程、定義佢被允許嘅工具、接駁佢嘅 critique 關係、寫好佢嘅配置並註冊佢、撰寫佢嘅版本化提示、先寫好佢嘅失敗測試、透過 factory 將佢實例化而唔係寫訂製邏輯、審查佢，再喺用到佢嘅工作流程入面註冊佢。個計劃亦點名咗一啲要拒絕嘅反模式，例如冇評分準則嘅 agent、直接調用 provider 而唔係經工具介面嘅 agent、改動另一個 agent 產物而唔係發出 critique 嘅 agent，以及自我品質係無法量度嘅空泛文字嘅 agent。

講到測試同埋評估，個計劃堅持，因為個系統本身就係一部評估引擎，佢自己嘅測試套件就一定要做得堪稱典範。一共有五層。單元測試係純邏輯、完全 mock、確定性而且快速，包括為啲契約做嘅屬性測試。契約測試會為每一個共享模型嘅 schema 影低快照，而一個移除或者重新命名欄位嘅改動，除非經過適當嘅版本控制同埋批准，否則會令到建構失敗。整合測試會對住真實嘅後端服務配 mock 生成器運行，涵蓋 graph 執行、殺死同恢復嘅持久性、事件重播、關卡中斷、契約傳遞、critique 路由，同埋預算強制執行。行為同埋 golden set 評估會將三層網應用喺系統自己身上，用凍結同埋釘死嘅 judge、一個模擬受眾，同埋一個每晚以及喺每次相關改動時運行嘅框架。對抗同埋安全測試會持續發動攻擊，例如深偽濫用、提示注入、越獄、誹謗同偏見，目標係攻擊成功率喺百分之一或者以下。而端到端測試就涵蓋瀏覽器入面嗰啲關鍵用戶旅程。持續整合金字塔會喺每次改動時運行嗰啲快速檢查，而喺每晚就運行嗰啲繁重檢查。

講到可觀測性、成本、安全同埋合規，每一個 agent 運行、工具調用、模型調用同埋關卡決策都會變成一條 trace span，所以一個製作會由簡報到交付形成一棵 trace 樹。指標會流去儀表板，事件溯源嘅 bus 同埋工作流程歷史令到任何製作都可以被重播，而日誌都係結構化並且以識別碼互相關聯。成本會逐個調用計量入預算事件同埋儀表板，喺違反時硬性停止同埋升級處理，並且有一個成本優化器將路由保持喺最佳成本同品質嘅前沿。安全同埋合規會由一個 compliance agent 強制執行，佢喺每一條發佈路徑上都可以攔截，喺任何肖像或者聲音生成之前都要求一條同意鏈，為百分之一百嘅可發佈產物簽署，並且具備完整嘅來源紀錄同埋審計。個計劃定義咗一個唔可以討價還價嘅發佈謂詞。一件產物只有喺通過規格層、喺評分準則上至少攞到八十五分、達到偏好門檻、通過全部六項交付檢查、過到合規、已經簽署，並且喺預算之內嘅情況下，先算係可以發佈。成個條件會包裹喺一個函數入面，而 gatekeeper 就只會調用嗰一個。

講到部署，一共有三個環境，即係用 mock provider 嘅開發環境、用合成負載嘅暫存環境，同埋用真實 provider、完整密鑰，以及一個自動擴展圖形處理器池嘅生產環境。啲管道會喺每次改動時運行驗證、契約同 golden 檢查，喺主分支運行完整整合，並且運行嗰套繁重嘅每晚套件。慣例包括按里程碑劃分範圍嘅結構化提交、短命嘅分支、唔可以直接推送去主分支，以及每一個改動都要經過一個有審查嘅請求。

講到數據、模型同埋提示管理，每一個提示都會被版本化並且記錄入來源紀錄，而提示嘅改動喺晉升之前要經評估把關。模型會按每個 agent 嘅政策釘死，而升級要經評估把關，唔容許任何退步。種子、風格參考，同埋參考幀庫都會被版本化以保證可重現性。Golden 固定裝置會被凍結並且審查，而更改一個預期輸出就需要理據，因為咁可能代表評分準則漂移咗。美學檔案受同意管治並且簽署，而評估數據集就包裹喺一個共通介面後面，並且長期追蹤基線。

個計劃包含一份風險登記冊，點名啲風險同埋佢哋嘅緩解措施。大嗰啲包括喺廣泛建立 agent 之後先發現架構缺陷，由垂直切片緩解。所有 agent 之間嘅契約漂移，由凍結契約同埋快照測試緩解。失控嘅成本，由計量、預算關卡、測試入面用 mock，同埋一個即時冒煙上限緩解。喺 graph framework 同 workflow engine 之間嗰個邊界產生嘅混亂，由一個有案可查嘅決策同埋殺死及恢復測試緩解。嘈雜嘅 judge 評分令到關卡不穩定，由凍結釘死嘅 judge 緩解。獎勵駭客產出靚但係空洞嘅輸出，由 aesthetics agent 嘅反駭客層同埋喺低置信度時嘅人類審查緩解。Provider 中斷、同意同埋知識產權違反、上下文膨脹令到 coding agent 退步、經攝取內容而嚟嘅提示注入、擴展瓶頸、不穩定嘅測試，同埋新 agent 其實冇填補任何真實缺口嗰種範圍蔓延，每一個都有佢自己嘅緩解措施。

跟住就到嗰張一百點加固清單本身，亦即係「諗一百次」呢個指令嘅字面同埋結構化形態，組織成十個主題、每個主題十項檢查。主題一係契約同埋 schema 完整性，確保每一條訊息都有型別、每一份交接都有填好、快照守護住啲模型，並且冇任何套件喺本地重新定義一個共享契約。主題二係編排同埋狀態，涵蓋殺死同恢復、扇出同扇入嘅正確性、死鎖偵測、逾時、有案可查嘅框架邊界、真正會阻塞嘅關卡、冪等性、確定性重播、背壓，同埋優雅降級。主題三係 agent 正確性，確保所有 agent 都係嚟自 factory、通過佢哋嘅規格同評分準則檢查、同佢哋嘅 critique 邊完全吻合、強制執行佢哋嘅工具許可清單、為佢哋嘅精煉迴路封頂、保存佢哋嘅教訓、永遠唔改動另一個嘅產物、用可量度嘅自我品質，並且尊重合規攔截。主題四係品質網，確保只用嗰一個發佈謂詞、啲確定性驗證器係完整嘅、啲 judge 係釘死同穩定嘅、模擬受眾嘅規模啱晒、全部六項交付通過都把關、偵測到連續性漂移、美學評分被分解、獎勵駭客防禦處於啟動狀態、無障礙有把關，並且一個植入嘅缺陷會被可靠咁捕捉到。

主題五係成本同埋效能，涵蓋計量準確性、預算強制執行、最佳路由、快取、延遲目標、自動擴展、儲存分層、工作負載分離、即時冒煙嘅預算上限，同埋並行負載。主題六係安全、保安同埋合規，涵蓋紅隊成功上限、注入防禦、同意鏈、完整簽署、各項監管清單、密鑰保護、基於角色嘅存取、依賴掃描、披露，同埋對個人資訊嘅小心處理。主題七係可觀測性同埋可操作性，涵蓋每個製作一棵 trace 樹、啲儀表板、結構化日誌、可重播性、警報、快速退步偵測、操作手冊、備份同復原、災難復原演練，同埋為安全推出而設嘅功能旗標。主題八係前端同埋人類體驗，涵蓋完整嘅瀏覽器工作流程、快速準確嘅即時更新、冇直接調用介面、重新連線處理、批准時簽署、即時儀表板、瀏覽器測試旅程、存取控制、用戶介面改善待辦清單，同埋按規模自適應嘅工作流程。主題九係建構流程同埋 coding agent 嘅衛生，涵蓋當下精簡嘅記憶檔案、有定義嘅子 agent 同指令、有強制執行嘅 hook、一份完整嘅決策日誌、一份準確嘅進度檔案、乾淨且劃分範圍嘅提交、每次提交都有一個綠燈嘅驗證關卡、有實踐嘅上下文衛生，同埋有遵守嘅測試先行紀律。而主題十係端到端系統驗證，涵蓋全部十個原型喺 mock 上運行、劇情片演練操練全部一百一十四個 agent、可驗證嘅來源紀錄、有效嘅多渠道交付、重新訓練迴路嘅閉合、可展示嘅優化改善、creative 同 aesthetics 迴路產出更好嘅候選、來源評級嘅 research、一次受預算約束嘅成功即時冒煙，最後就係，一個全新嘅工程師可以單憑呢個計劃同埋啲規格建構出嚟，唔需要任何口耳相傳嘅知識。完成嘅意思係一百項裡面有一百項打咗剔，每一項旁邊都有證據。

個計劃以一段排序總結作結，將每一個里程碑對應到佢嘅驗收關卡同埋佢嘅主要規格，並且點名嗰條由啟動開始、經過編排、gateway、智能層、垂直切片、闊度、meta agent、支援，再到加固嘅關鍵路徑。介面工作可以喺早段並行，而增強層就喺接近尾聲嗰度插入。槓桿最高嗰個檢查點係垂直切片關卡，因為佢將架構風險轉化成一個經過驗證嘅基礎。喺呢度全力以赴，意思係開頭時深度行先過闊度，然後經 factory 做機械式嘅闊度擴展，跟住做遞迴式嘅品質，再做嗰張字面上嘅一百點掃蕩。成個計劃嘅設計，係要令到一個缺陷喺佢最有可能被發現嗰一刻，剛好亦係最平嘅去修。

仲有啲附錄，附有可以複製貼上嘅起步材料、一個專案記憶檔案嘅範本、子 agent 嘅定義、斜線指令嘅定義、一個關於權限同 hook 嘅設定檔、一個要釘喺每一個改動請求入面嘅完成定義，同埋一份關鍵術語嘅詞彙表。

而最後嗰句說話，用一句就捕捉到成份信念。喺鋪闊度之前，先建立平台同埋一片完美嘅切片，然後就等一個 factory 同埋一個遞迴式品質網去做擴展。契約最先被凍結，咁啲 agent 就唔可以分歧。垂直切片將系統最大嗰個風險，亦即係太遲先發現架構缺陷，轉化成一個平、早，而且可以證明嘅檢查點。品質係遞迴嘅，所以嗰個審判影片嘅系統，喺自己身上都要通過同一套標準。而「諗一百次」呢個指令唔係修辭。佢就係嗰個字面上嘅一百點關卡，企喺「喺我部機度行得到」同埋「生產環境」之間。畀 coding agent 嘅指示好簡單。由里程碑零開始，然後開工。
