第三條支柱係基礎嘅推理同行動範式同佢嘅各種增強。核心將一個思考，即係一段推理軌跡，同一個行動，好似工具呼叫、委派或者完成，交錯埋一齊，跟住係一個有根據嘅觀察。喺呢個基礎之上，個設計摺入反思式嘅口頭自我批判、可選嘅生成同排名多條候選軌跡、一種喺推理步驟本身就加強根據嘅技巧，做法係重新錨定喺最新嘅觀察同原本嘅目標，以及明確嘅計劃同執行範式，配合可選嘅樹狀搜尋元素。佢由頂尖供應商借來伺服器端嘅推理迴圈，由模型決定工具、喺內部執行、然後迭代到有最終答案，以及平行專家配一個領導者嘅模式。呢度嘅混合做法，係由一個客戶端編排者掌管全域狀態、追蹤同驗證，同時將深入研究嘅子問題委派畀伺服器端嘅多智能體系統，而編程子任務就用本地專家，或者用帶住複製狀態切片嘅隔離平行進程嚟做。

經過咁多番重新思考，最終嘅架構決定就好清楚。迴圈風格係一個受控、自訂嘅推理同行動迴圈，以雜湊為本嘅迴圈偵測同斷路器做基礎，再喺上面疊上階層，由一個編排者去決定係委派、呼叫工具、綜合，定係完成。中央控制勝過協調嘅混亂。狀態物件持有任務規格、歷史、todo 清單、計劃、短期同長期記憶、版本登記冊、追蹤器、預算，以及用嚟做迴圈偵測嘅已見雜湊集合。記憶倚靠結構化嘅 todo 同關鍵事實做低 token 嘅主力，有壓力時做摘要，配一個可選嘅向量庫做語意檢索，而子 agent 永遠只會收到帶住來源嘅情境切片。模型呼叫經過一個統一嘅客戶端，支援多個供應商，永遠帶住一個系統 prompt、適當嘅 few-shot 例子，同一個嚴格嘅輸出結構。工具住喺一個有驗證嘅登記冊，配一個安全執行嘅包裝。混合整合有一個專門嘅包裝，會送出一個狹窄嘅子目標連同成功準則同已啟用嘅工具，並且將結果記錄成一個特別嘅觀察。自我演化嘅範圍隨階段擴大，由 prompt 去到工具碼，再去到 agent 配置。而系統會自食其力，喺開發期間用個 harness 喺保留軌跡上改良佢自己嘅 prompt 同驗證器。

而家去到系統架構同高層次嘅流程，佢行過六個階段，並且用認知層加固咗。第零階段係初始化，以規格驅動，連認知設定。佢將指令解析成一份經過驗證嘅任務規格，包括認知檔案，跟住行一個規格驗證器同批判者，去檢查完整性、含糊、角色清晰度同終止條件。然後就到強制嘅 premortem：假設規格同計劃慘烈噉失敗咗，找出最主要嘅原因同緩解措施，再將佢哋摺返入活的規格、成功準則、todo 同質素關卡。然後 Cynefin 分類將任務標籤為簡單、繁雜、複雜或混沌，並且用呢個去自動配置迴圈，簡單同繁雜嘅工作偏向快速路徑，複雜同混沌嘅工作就行完整嘅審慎路徑連同重度反思。Agent 狀態建立時帶住做 recognition-primed 決策嘅模式庫，同一個快速或完整嘅當前模式。可選地，一個規劃者會生成一個高層次計劃同一份 todo 文件，對住規格同 premortem 風險做驗證。最後，佢決定架構，係平面、階層定混合，再設定初始模式。

第一階段係連認知模式選擇嘅核心受控迴圈。喺每次迭代開始嗰陣，或者喺一個重大觀察之後，系統決定佢嘅運作模式。如果 Cynefin 容許，而模式庫又有一個高相似度嘅匹配，加上後設認知嘅信心高，佢就行快速辨識路徑：一個輕量嘅思考去引用匹配到嘅軌跡、最少嘅 token，直接去到行動。否則佢就入完整審慎模式，有一個詳細嘅思考去重新錨定喺最新嘅觀察，連同完整關卡，同時有一個輕量嘅平行後設認知監控器，去掃描偏見、對住準則檢查進度、留意情境漂移，並且把一把信心嘅脈，隨時準備強制換模式或者提早重新規劃。佢由摘要過嘅歷史、關鍵事實、todo、規格、最新觀察同相關嘅模式庫條目去建構情境。模型產出一個嚴格嘅決定：先有一個帶後設認知罩面嘅思考，然後一個行動類型，然後一個酬載。有一個迴圈檢查，會雜湊近期嘅行動同觀察，如果之前見過就強制重新規劃或終止。執行經過安全嘅工具呼叫或者安全嘅委派。每一個結果都變成一個結構化觀察，而家亦都記錄用咗邊個模式。每一個事件都附加入歷史，todo 同進度都更新，模式庫就喺高質素結果出現時更新。輕量嘅反思隔一段時間或者出錯時做，連同一個關於模式幾有效嘅備註。斷路器逐個工具同角色噉運作，而終止可以嚟自達到準則並通過驗證、撞到步數或預算上限、一個明確而經驗證嘅完成、一個要升級嘅不可挽回情況，或者喺子準則上一個強勁嘅快速路徑成功。

第二階段係階層式委派。編排者分解工作，再由登記冊揀或者實例化一個專家，好似研究員、編程員、驗證員、報告員、工具生成員、瀏覽器 agent 或分析員。佢建立一個狹窄嘅子任務規格，帶住目標嘅子集、成功準則、一個情境切片同來源，然後調用個子 agent，可以係佢自己嘅受控迴圈，或者一個伺服器端嘅多智能體委派。個子 agent 回傳一個結構化觀察，連同佢軌跡嘅摘要冒上嚟做審計，而父層就記低佢、驗證並整合佢、更新全域計劃，再決定下一步。

第三階段係整合同質素關卡。一個聚合器收集啲觀察同計劃進度。一個協調者同報告員去合併、去重、交叉參照同解決衝突，做法係引用來源同版本，產出一份統一嘅草稿。驗證器對住成功準則畀分、檢查幻覺、漏洞同驗證失誤，並且建議修正，全部以結構化輸出。如果過唔到關卡，就觸發精煉，重新規劃一個分支、重新委派，或者自我編輯。如果過到，就去到打磨或最終輸出。

第四階段係反思同自我演化，係進階嗰部分，結合 after-action review、double-loop learning、五個為什麼同多角度批判者。喺里程碑或者結尾，完整嘅軌跡同模式歷史會去到自我演化器。強制嘅 after-action review 回答四條問題：本來應該發生乜、實際發生咗乜（包括快速對完整嘅模式統計同模式匹配）、點解會噉（連一個深入嘅五個為什麼嘅一趟，同石川圖分類，橫跨好似 prompt 同方法、模型同工具、資料同觀察、情境同環境、狀態同記憶、驗證，以及人類規格呢啲維度），最後就係下一步點做，化成具體嘅改動。然後 double-loop 嗰層質疑主導嘅假設：質素嘅定義係咪太鬆？角色界線係咪容許咗漂移？成功準則或者認知預設本身應唔應該演化？只有經過後設批准嘅改動先可以推進。根本原因會帶住對失敗模式同認知框架嘅意識去診斷，會提出針對性嘅改動，去改 prompt、工具碼、角色定義，甚至認知預設或者新嘅批判者模式，喺有配置嘅情況下用 Paul-Elder 標準或者一次紅隊攻擊。改良會喺保留嘅例子上或者用重播去驗證，而更好嘅版本就帶住血緣同一個 after-action 嘅理據去登記，模式庫就為將來嘅辨識去更新。全部都有界限喺三輪，快速路徑嘅軌跡就用一個較輕嘅檢討。

第五階段係終止同輸出：一個依規格要求格式嘅最終綜合、持久化完整嘅軌跡、版本同指標、一個可選嘅事後反思摘要，以及喺高風險關卡或預算耗盡時嘅人類介入鈎子。

核心嘅資料模型應該最先生成，因為佢哋就係契約。有一個任務規格，承載目標、成功準則嘅清單、限制、輸出格式、質素關卡、一個可選嘅初始計劃、元資料，以及認知檔案連同佢嘅開關，去控制快速路徑、反思風格、批判者模式、Cynefin 分類同最大反思輪數。有一個結構化觀察，承載狀態、資料、一個摘要、一個零到一之間嘅信心、問題、下一步建議、來源、一個軌跡 id，同用咗嘅版本。有一個軌跡事件，承載步數、時間戳、思考、行動類型、酬載、觀察、token 用量同版本。仲有 agent 狀態，將任務規格、歷史、todo、計劃、短期同長期記憶、模式庫參照、已見雜湊集合、預算、版本、追蹤器、當前模式、Cynefin 情境，以及 after-action review 成品嘅歷史，全部綁埋一齊。

要實作嘅關鍵模組包括：核心迴圈、一個帶指標同逐個工具實例嘅增強斷路器、一個調校去捉特定失敗模式嘅驗證器、一個有規劃、路由、子 agent 工廠同整合器嘅編排者、一個由軌跡去診斷並提出同驗證改動嘅自我演化器、一個畀工具、環境同 agent 抽象嘅最小協議模組連版本管理器、一個畀伺服器端多智能體呼叫嘅整合包裝、記憶同追蹤模組，以及一組有版本嘅 prompt 檔案，喺批判者 prompt 入面引用啲失敗模式。專就混合整合嚟講，當行動需要研究委派，或者子規格複雜度高嗰陣，系統就會呼叫多智能體系統而唔係一個本地子迴圈，有一個可配置嘅門檻，永遠捕捉來源同 agent 數目，而如果呼叫失敗或者斷路器打開，就退返去一個本地子 agent 或者直接工具。佢亦可以模仿建構工具嘅做法，輸出計劃，並且喺獨立分支上用帶住隔離狀態副本嘅平行子 agent。

分階段嘅路線圖畀咗 coding agent 一條清楚嘅路。第零階段建立骨架、資料模型、一個基本嘅模型客戶端、嚴格結構嘅輔助器，同一個喺模擬工具上行得而唔崩潰嘅簡單迴圈；佢嘅關卡係個迴圈喺一個玩具任務上行十步、永遠產出有效嘅觀察、偵測到注入嘅重複，並且捉到明顯嘅規格問題。第一階段建立完整嘅受控迴圈，連迴圈偵測、完整嘅斷路器、安全執行、進度追蹤、連模式匹配嘅快速辨識路徑、輕量嘅後設認知監控器、輕量嘅反思、一個帶驗證器關卡嘅明確完成、結構化嘅狀態同積極嘅摘要，並且喺重複、歷史流失同過早或不完整終止上嘅失敗注入測試都要通過，快速路徑亦要正確噉觸發。第二階段加上編排者同一個可插拔嘅子 agent 登記冊、本地委派路徑、整合器同報告員，以及一個多層次嘅驗證器，喺一個帶委派嘅多步任務上端到端噉證明到。第三階段加上版本管理器、最小協議結構、自我演化器同伺服器端多智能體客戶端，自我演化喺三個相似任務上顯示出可量度嘅改善，委派亦端到端噉行得到。第四階段係打磨、例子、生產加固同自食其力：一個有認知路由嘅深入研究 agent、一個有規劃同子 agent 嘅編程專案 agent，同一個演化自己驗證器、prompt 同認知檔案嘅自我改良 harness，加上一個沙盒化嘅程式碼工具、一個命令列介面同可選伺服器、一個包含認知功能測試嘅完整測試套件、帶模式同檢討覆蓋層嘅可觀察性輸出、成本儀表板同文件。貫穿始終嘅使命：每個模組之後，生成程式碼、行一個內部批判者、修正問題、對住關卡重新驗證，然後先至前進。

講到生產加固、安全、可觀察性同可擴展性，啲主題一路貫穿。可靠性嚟自斷路器、帶退避嘅重試、結構化嘅錯誤觀察、預算執行同以進度為本嘅提早退出。安全嚟自將工具執行沙盒化、驗證同清洗所有模型生成嘅引數、最小權限嘅工具存取，以及對迴圈模式嘅異常偵測。可觀察性將追蹤器當成第一等，記錄每個事件連可選嘅完整情境快照、版本、token 計數、時序同子呼叫，可以輸出嚟由任何一步重播，並且可選地整合入現有嘅日誌。成本同可擴展性嚟自逐階段預算、只喺獨立分支上嘅平行、醒目嘅摘要訊號同會話隔離。而可擴展性嚟自可插拔嘅模型後端、一個有結構驗證嘅簡單工具登記冊、一個有角色 prompt 嘅子 agent 工廠、可插拔嘅記憶後端、可更換嘅演化策略，以及畀狀態機框架同遠端編排嘅轉接器。

測試策略好詳細，並且當成關鍵嚟睇。單元測試涵蓋結構驗證、迴圈雜湊嘅正確性、斷路器完整嘅狀態機（包括半開恢復），以及驗證器喺邊界情況上嘅解析同邏輯。整合測試喺玩具任務上行完整迴圈，並且演練階層式委派同混合路徑。失敗注入測試模擬每一個主要失敗模式，並且斷言及早偵測、正確緩解同冇無聲失敗。一個迷你基準衡量成功率、效率，以及三輪之前同之後嘅演化差距。基於屬性嘅測試將隨機嘅有效同無效規格同觀察掟畀系統，並且斷言不變量，好似永遠係結構化輸出、合理嘅信心值、受尊重嘅預算同一致嘅版本。而一個自食其力嘅演化測試，會用真實軌跡喺驗證器或規劃者 prompt 上行自我演化，並且驗證改良版本喺保留資料上分數更高，而又冇喺基礎任務上退步。使命好簡單：一個階段嘅測試未通過之前，唔好標佢做完成，而且要連同程式碼一齊生成測試。

啲參考將呢一切落實喺真實嘅工作上：多智能體失敗分類法、工具—環境—agent 嘅編排協議、基礎嘅推理同行動論文、一系列反思同規劃嘅增強、近期嘅 agent 調研、頂尖供應商嘅生產文件、佢所建基嘅第三版規格，以及嗰張排好名次、列出四十個傳統人類思維模型連佢哋採用分數同整合理據嘅表。

最後，交接。呢份規格係完整而可行動嘅，建議係即刻由第零階段嘅骨架同核心模型開始，因為嗰啲模型就係契約。仲有一個建議畀 coding agent 嘅起手 prompt：完整噉讀相關嘅規格，建立套件骨架連核心模型（包括認知檔案、模式庫同當前模式）、建立一個基本嘅模型客戶端、一個 Cynefin 分類器嘅雛形，同第零階段嘅一個 premortem 步驟，以及一個帶快速路徑同後設認知模式選擇嘅最小可運作受控迴圈，去通過早期嘅驗證關卡，用嚴格嘅結構，連失敗注入同認知功能測試嘅雛形，貫穿始終維護一份 todo 文件，並且對每個模組都施加一個批判者。核心迴圈穩陣之後，逐階段推進，並且喺自食其力嘅階段，用所建構嘅系統去演化佢自己嘅 prompt 同驗證器。預期唔會有要澄清嘅問題，因為份規格係自足嘅；如果出現含糊，就由內部批判者去解決，或者帶住具體嘅軌跡輸出去升級。預期嘅交付成果係一個功能完整、經過測試、有文件嘅套件，連同例子同一個命令列介面，任何人都可以匯入並運行，去做可靠、cognitive-enhanced、階層式嘅 agent 工作流，並且對混合整合、自訂工具、自我演化同認知配置有清晰嘅擴展點，以及完整支援帶模式同檢討覆蓋層去重播軌跡。簡而言之，將佢起到生產級、令認知層成為一個第一等而可配置嘅強項、喺每個模組都用批判者去迭代、等系統改良自己，並且將佢變成先進、與人類對齊嘅 agent harness 嘅基石。佢已經準備好執行。
