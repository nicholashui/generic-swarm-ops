Three cooperating sub-trees under one repo root:
- `scripts/` — Node.js entry points (`doctor.mjs`, `create-project.mjs`, `sync.mjs`, `review.mjs`, `security.mjs`, `source-audit.mjs`, `export-openapi.mjs`) plus Python generators (`_gen_*`, `_inject_*`). Shared utilities live in `scripts/lib/` (`fs-safe.mjs`, `git.mjs`, `manifest.mjs`, `report.mjs`, `scaffold.mjs`) and are imported by both JS and Python via relative paths.
- `scripts/business/` — domain-specific orchestrators: schema validation (`schema_validate.py`, `validate-business.mjs`), migration/task generation (`migrate_va_corpus.py`, `run_migration_tasks_fleet.py`, `*_migration*`), evaluation harness (`eval-harness.mjs`, `harsh_score_v1.py`), governance/security checks (`governance-check.mjs`, `security-controls.mjs`), and evolution sandbox tooling. Business helpers are grouped under `business/lib/` (`risk-tiers.mjs`, `schema-lite.mjs`, `business-report.mjs`).
- `scripts/adapters/` — harness adapters (`grok.mjs`, `trae.mjs`) share a single source of truth in `adapters/shared.mjs` which defines `BUSINESS_AGENTS`, `HARNESS_COMMANDS`, `PROJECT_SKILLS`, and markdown builders; each adapter translates these into `.grok/*` / `.trae/*` config trees.
- `skills/` — flat skill catalog declared in `skills/manifest.json`; each skill is a folder containing a frontmatter-bearing `SKILL.md`. The manifest lists top-level categories (`planning`, `implementation`, `testing`, `review`, `security`, `memory`, `lifecycle`) consumed by `adapters/shared.mjs`'s `PROJECT_SKILLS` list.
- `hooks/` — hook registry in `hooks/manifest.json` declares Grok `SessionStart` hooks and external references to `external/sources/*/hooks`; the actual hook script lives at `hooks/scripts/session-start-reminder.mjs`.
Dependency direction is one-way: adapters + business scripts import `lib/` and `adapters/shared.mjs`; `skills/` and `hooks/` are pure data consumed by the scripts, never importing them back.
