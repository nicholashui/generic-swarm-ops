The module is a documentation-only portfolio organized by concern rather than runtime layer:
- `backend/` — 24 numbered sub-functional specs (01–24) covering platform charter → scaffold → persistence → API envelope → auth/RBAC/users/orgs → agent/tool registries → workflow definition/execution engine → governance/policies/human gates → audit/logging → knowledge base/memory → evaluation/process intelligence → streaming/health → evolution sandbox/self-improvement → DNA safety/security hardening → testing strategy. Each spec folder holds `requirements.md`, `design.md`, `tasks.md` plus quality-score and traceability indexes.
- `frontend/` — 20 numbered sub-functional specs mirroring the same lifecycle: charter → Next.js scaffold → design system/OpenDesign → app shell/navigation → auth/session UI → permission-aware UI → typed OpenAPI client → domain pages (dashboard, agents/tools, workflows, runs, approvals, knowledge/memory, evaluations, audit logs, settings) → evolution/improve pipeline → accessibility/error states → security/performance/testing ops profile.
- `structure/` — 17 core-system specs (charter → artifact repo → intake/risk router → governance tiers → security controls/tool broker → process intelligence → knowledge elicitation/decision cards → hybrid memory → tiered retrieval → workflow DNA → bounded execution → human gates/audit → evaluation harness → evolution sandbox → agent roster/control roles → HAI rules → phased rollout), with additional cross-cutting files (`IMPLEMENTATION_STATUS.md`, `GAP_ANALYSIS_IMPLEMENTATION_VS_TASKS.md`, `PR_PRIORITY_LATTICE_CHECKLIST.md`).
- `improvement/` — six waves (wave-0 through wave-5) of incremental capability delivery, each following the mandatory SDD quartet (`requirements.md`, `design.md`, `tasks.md`, `completion-report.md`) plus a chain-run report; tracks ALC gate, spine/viral-hook DNA, coevolution, multi-pack load, full roster wiring.
- `special/` — generated SDD adoption plans for 17 special skills sourced from `study/`, produced by `scripts/business/generate_special_skills_plans.py`.
- `migration/` — video-domain implementation order list, per-agent markdown/PDF artifacts, and accessibility notes.
- Cross-cutting gap analyses at the root (`gap_analysis_for_backend.md`, `gap_analysis_for_frontend.md`, `gap_analysis_for_structure.md`) link as-built code to these specs.
Dependency direction is strictly top-down within each track: charter → scaffold/foundations → domain features → safety/wrappers → proof/testing, with explicit dependency sketches in each README.
