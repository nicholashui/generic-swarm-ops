The frontend is a Next.js App Router project with two top-level route trees: `/` (root layout in `src/app/layout.tsx`) serves the public landing page, while `/app/*` is gated by both server-side middleware (`middleware.ts`, redirects unauthenticated requests to `/login` unless `NEXT_PUBLIC_DEMO_MODE=true`) and a nested `src/app/app/layout.tsx` that calls `getSessionUser()` and wraps children in `AppShell`. All domain UI lives under `src/components/domain/`, composition shell under `src/components/layout/`, and shared primitives under `src/components/ui/`. Cross-cutting concerns are centralized in `src/lib/` (api, auth, config, data, errors, formatting, forms, realtime, routes) and consumed via React hooks in `src/hooks/` plus Zustand stores in `src/stores/`. The UI is typed against a single OpenAPI schema: `pnpm api:generate` exports `openapi.json` from the backend and produces `src/lib/api/generated/openapi.d.ts`, which all API consumers import. Design tokens and Tailwind v4 theme live in `src/design/` and are applied through the root `globals.css`.
