- Each domain has a parallel trio of files: `domain/<x>/models.py` (Pydantic schemas), `services/<x>_service.py` (orchestration), and `api/v1/routes/<x>.py` (endpoint wiring), keeping request/response shape separate from persistence.
- Infrastructure backends expose a small abstract base class (e.g. `infrastructure/vector_store/base.py`, `infrastructure/object_storage/base.py`, `infrastructure/llm/base.py`) with concrete implementations in sibling modules selected at runtime via settings.
- Configuration is centralized in `core/config.Settings`, a frozen dataclass whose fields read `GENERIC_SWARM_*` / `OPENAI_*` / `NEO4J_*` environment variables with typed defaults, consumed everywhere via the module-level `settings` instance.
- Authentication flows return typed exceptions from `runtime.RuntimeErrorBase` subclasses (`NotFoundError`, `PermissionDeniedError`, `ApprovalRequiredError`, `RateLimitedError`) rather than raising raw HTTP errors, letting `api/errors.register_error_handlers` serialize them uniformly.
- Collections in the runtime store are accessed through `runtime.store.collection(name)` which auto-creates empty lists for unknown keys, so callers never check existence before appending.
- Request-scoped state (request id, current user) is propagated with Python `contextvars.ContextVar` (`CURRENT_REQUEST_ID`) instead of global mutable variables, set by the top-level HTTP middleware.
