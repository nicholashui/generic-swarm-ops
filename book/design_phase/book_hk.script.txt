歡迎嚟到呢個關於通用蜂群業務操作系統 (Generic Swarm Business Operating System) 嘅詳細介紹。今日我會同大家由架構到實作、由核心原則到產品標竿，全面講解呢個系統，等你聽完之後對成個倉庫有深入理解。

好，首先要講呢個系統係咩嚟。簡單嚟講，佢係一個受治理、自我改進嘅多代理系統 (multi-agent system)。佢做四樣主要嘢。第一，學習企業實際運作方式，唔單止睇文件同訪談，最重要係從真實事件日誌 (event logs) 學。第二，將學到嘅知識蒸餾成可重用規則、技能、工作流程 (workflows) 同手冊。第三，通過有邊界、可審計嘅代理工作流程執行工作。第四，喺沙盒 (sandbox) 入面演化呢啲工作流程，絕對唔會直接改生產環境 (production)。

再嚟，點解要咁設計？關鍵係自主權 (autonomy) 概念。喺呢個系統入面，自主權唔係預設畀你，而係要賺返嚟。一個工作流程由只可以觀察開始，慢慢累積通過嘅評估 (evaluation)、回滾計劃 (rollback plan) 同人類簽核，先一步步爬上自主梯。將呢個系統同無約束人工智能群體區分開嚟嘅地方係：圖 (graph)，而唔係模型 (model)，掌握權威。

跟住核心原則有七項。第一，證據優於意見，要從真實追蹤記錄學，唔好淨聽人講。第二，有邊界嘅自主，每個動作都有風險等級 (risk tier)、權限範圍，同需要時嘅人類關卡 (human gate)。第三，一切都可測試，冇代理、提示 (prompt) 或者工作流程可以喺唔通過評估之下入生產。第四，沙盒演化，演化引擎只提出建議，絕對唔直接改生產。第五，始終有來源追溯 (provenance)。第六，可逆性優先。第七，以人為中心，顯示信心、顯示證據，令修正容易。呢啲原則用明確優先順序表達：安全先，然後可審計性，跟住正確性，再到效率，最後先到自主性。如果令工作流程更自主會降低可審計性，可審計性贏；如果更快檢索會削弱安全，安全贏。

而家講下高層架構。系統最頂係入口加風險路由器 (intake and risk router)。每一個事件或者請求，無論新工單、應用程式介面 (API) 呼叫定排程作業，都會先去呢度做風險分類，再交畀業務協調器 (Business Orchestrator)，一個擁有全局目標同路由工作嘅狀態圖控制器。從協調器開始，工作流經六個協作層。第一層流程智能 (process intelligence)，由事件日誌學真實工作流程。第二層知識同記憶 (knowledge and memory)，處理檢索增強生成 (RAG) 知識、差異化記憶同來源追溯。第三層執行，用工作流程基因 (workflow DNA) 描述嘅有邊界狀態圖。第四層演化，喺沙盒提出同測試變體，所有變體喺受閘晉升前都係 sandbox_only。第五層治理 (governance)，風險等級、審批規則同審計要求。第六層安全，對抗測試、提示注入防禦同爆炸半徑控制。成個架構關鍵洞見係將權威同智能分開：語言模型 (LLM) 喺圖節點入面提供智能，而圖本身作為確定性可檢查代碼提供權威，強制狀態、權限同人機迴圈關卡。安全同治理喺模型之外確定性執行，因為系統提示本身唔係安全控制。

接落嚟入門層。業務系統存在之前，倉庫必須係合法可執行嘅代理 harness 專案。入門層會下載、審計、篩選同同步參考來源到 Trae IDE 同 Grok Build 雙 harness 工作空間，用 npm run sync 生成 .trae 同 .grok，包括專案規則、自定義代理、技能、命令、模型上下文協定 (MCP) 配置同支援文件。下載落嚟嘅倉庫係不受信任參考輸入，下載器只會複製同檢查元數據，絕唔執行安裝腳本或者 npm install，亦唔會寫出專案根以外。只有經過篩選審計歸因嘅材料先可以複製到第一方目錄，影響高要人類批准。

第一層流程智能堅持由實際操作追蹤學習，例如工單、客戶關係管理 (CRM) 操作、日曆、電郵、審批、文件編輯、應用程式介面呼叫同完成記錄。流程挖掘會將日誌轉成可發現工作流程模型、合規檢查同瓶頸分析。有專門代理做流程挖掘、任務挖掘、合規、瓶頸同因果改進。發現嘅流程會餵畀知識蒸餾、評估系統同演化沙盒。

第二層知識、蒸餾同混合記憶。專家知識好多係隱性，系統用多方法捕捉，包括影子模式、關鍵決策訪談、大聲思考、例外訪談、回顧同學徒模式，產出決策需求卡 (decision requirement cards)。記憶有八種類型，例如事件、情節、語義、程序、決策、例外、評估同來源記憶，而且記憶預設唔全局可用，查詢要考慮組織、用戶、代理、工作流程、敏感性同策略。檢索用成本分層，第零層向量搜尋最平，第一層圖關係推理精簡版，第二層分層摘要按需，來源層永遠在線。產品標竿 as-built 有 K1-lite 圖抽取同算子，以及 Cypher 或者 JSON 聯邦匯出，完整 LightRAG 同 Neo4j 生產網屬非目標。

第三層執行工作流程基因。執行唔會自由形式群體發生，而係有邊界狀態圖。工作流程基因要聲明步驟、明確代理、明確工具、記憶讀寫、護欄、驗證、回滾、適應度指標同審計寫入。驗證器會拒絕高風險不可逆步驟冇人類關卡，或者冇回滾計劃嘅生產工作流程。運行時將基因實現做帶關卡線性狀態機，每步前檢查取消、權限、治理，需要就請求批准，再執行代理或者工具，寫步驟輸出、串流事件同審計日誌。

第四層演化引擎最危險，所以有一條不可談判規則：演化管理器絕對唔可以直接變更生產，只可以提出變體、沙盒測試、同基線比較、請求批准、金絲雀 (canary) 部署、失敗自動回滾。變體喺受閘晉升前一律標記 sandbox_only。變體用明確加權函數評分，包括品質、安全、合規、效率、人類滿意度，同風險延遲成本懲罰。推廣要同時改善目標指標、冇安全合規倒退、通過回歸同對抗測試、有回滾計劃、完整審計日誌，同風險需要時有人類簽核。

第五層治理錨定 NIST 人工智能風險管理框架、ISO 四二零零一同歐盟人工智能法案。自主風險有六級，由觀察、建議、起草、執行可逆、執行加關卡，到受限要保證案例。第六層安全將攻擊面擴大到超越傳統應用安全，要同時參考大語言模型應用十大風險同代理應用十大風險，用確定性控制喺模型之外擋提示注入、工具濫用同記憶中毒。

評估系統包裹所有層，有金任務 (golden tasks)、回歸、對抗同歷史重放，重要失敗可以擋放行或者晉升。代理名冊列出業務協調器、流程智能代理、知識代理、執行代理、治理代理、安全代理、評估代理等同角色。人機互動規則要求顯示信心同證據，令修正容易，前端會把錯誤同請求識別碼 (request_id) 呈現畀操作者。

而家重點嚟啦，第十五章後端應用程式介面伺服器。後端唔係薄代理，而係受治理智能同控制層。前端永遠唔直接掂代理、資料庫、工作流程引擎、大語言模型供應商、向量儲存或者內部工具，一切都經應用程式介面。分層係路由薄、服務同領域邏輯厚、基礎設施適配器同提供者無關，橫切中介有認證、角色為本存取控制 (RBAC)、速率限制、冪等、請求識別碼、錯誤信封、指標同安全標頭。

技術棧 as-built 用 FastAPI 同 Python，主控制平面 Postgres 嘅 runtime_state JSONB，本機唔強制 Docker，JSON 檔做備份同種子。認證密碼雜湊用 PBKDF2，有 JWT 同應用程式介面金鑰，角色同資源動作權限，查詢一律組織識別碼範圍化。路由喺應用程式介面版本一路由下面，包括認證、使用者、組織、代理、工具、工作流程、執行連 cancel 取消、retry 重試、pause 暫停、resume 恢復、expire 逾期、審批、治理、知識、記憶、評估、審計、流程、演化、改進、迴圈、設定同健康探測。規格驅動開發包喺 planning/backend，BE-01 到 BE-24，有需求設計任務同交付程式路徑，落差報告 gap_analysis_for_backend 一百分。工具執行用本機適配器寫持久工具副作用 (tool_effects)，失敗封閉，外部即時客戶關係電郵軟件即服務屬非目標。演化同改進路徑強制 sandbox_only，生產基因啟用前要結構驗證器。

執行生命週期由前端 POST 開跑，後端驗證權限同治理，建立排隊執行，工作者逐步行，需要就等人批，再用伺服器推送事件 (SSE) 串流返前端。操作者可以經後端路徑做 pause、resume、expire，以及 cancel 同 retry。改進同演化永遠只限沙盒，晉升先可以離開 sandbox_only。

接落嚟第十六章前端營運主控台 (ops console)。前端係專業企業軟件即服務，唔好似通用人工智能 demo，要傳達信任可靠營運清晰安全專業速度控制同可觀測。前端只係呈現同互動，授權執行治理真相永遠喺後端。技術棧用 Next.js App Router、React、TypeScript、Tailwind、React Hook Form 加 Zod、由後端 OpenAPI 產生嘅型別化客戶端、SSE 做執行時間軸、Vitest 單元同 Playwright 煙霧測試、pnpm。重大版面應先走 OpenDesign 模型上下文協定，唔用得就文件化後備，as-built 有 open-design-reference。

資訊架構方面，公開有登入註冊忘記重設密碼同 accept-invite 接受邀請，已登入 app 下面有儀表板、代理、工具、工作流程、執行詳情、審批、知識、記憶、評估、流程、審計、演化同設定。側邊欄品質組有評估、流程同演化。命令面板用命令鍵或者控制鍵加 K。

最重要頁係執行詳情，WorkflowRunConsole 一開就連 SSE，顯示時間軸日誌工具呼叫同錯誤請求識別碼，操作者可以 cancel、retry、pause、resume、expire，即取消、重試、暫停、恢復、逾期。同頁有改進面板，反思提案評估金絲雀只打後端沙盒應用程式介面，變體保持 sandbox_only。演化頁 app evolution 只做沙盒族群同適應度。accept-invite 接受邀請表單會 POST 邀請接受路徑，成功寫工作階段入 app。設定用 UserAdminPanel 邀請同停用使用者，OrganizationSettingsForm 局部更新組織。審計前端唯讀。

前端子功能規格喺 planning/frontend，FE-01 到 FE-20，落差報告 gap_analysis_for_frontend 一百分。營運設定檔要把 NEXT_PUBLIC_DEMO_MODE 或者 DEMO_MODE 設做 false，對住 live FastAPI 加 Postgres。品質閘 as-built 包括 lint 零問題，form.watch 改做 useWatch 之後靜態檢查乾淨、型別檢查、單元測試同建置通過。

第十七章倉庫結構係三層蛋糕，入門層雙 harness、業務層、實作層，再加 planning 規格包。根目錄有 structure 同 structure_hk、backend 同 backend_hk、frontend 同 frontend_hk，.trae 同 .grok，business、backend、frontend、planning、book 等。planning/structure、planning/backend、planning/frontend 三套 SDD，加上 gap_analysis 三份報告。引導用 npm run bootstrap 跑醫生檢查、來源下載審計、安全、同步預覽、業務驗證同測試。

第十八章實作狀態，產品標竿 mark 大約一百分已關閉。E1 操作者路徑通過，自我改進同 K1-lite 已出貨，前端 residual 包括 accept-invite、pause 同 resume 同 expire、使用者邀請停用、組織局部更新都已出貨。Next.js ops console live 模式、真實表單、Run now、Improve、evolution archive、useWatch lint 乾淨、unit 同 build 綠燈。有結構後端前端三份 gap_analysis 各一百分。非目標包括外部即時 CRM 電郵 adapters、完整 LightRAG Neo4j 生產網、常駐 Playwright 伺服器、DGM 式主機自我改寫。

好啦，端對端演練講客戶入職。已簽署合約入入口，冪等鍵防重試重複開跑。風險路由器分類，業務協調器載入工作流程基因開執行。步驟一驗證合約，步驟二建立客戶記錄，步驟三配置帳單，步驟四寄歡迎包因為外部電郵不可逆要人批，前端審批頁批准後繼續。完成後評估、寫記憶，演化沙盒只觀察唔直接改生產，變體仍然 sandbox_only。

常見任務方面，加新工作流程要寫基因、驗證、註冊、做金任務同跑評估。捕捉專家知識用關鍵決策訪談同決策需求卡。提出變體要經演化沙盒同金絲雀，保持 sandbox_only 直到批准。安全事件跟事件手冊。審計用前端審計日誌唯讀搜尋。九十日部署由基礎建設、影子學習、受控副駕駛，到演化沙盒，成功標準包括引導全過、事件流入、決策卡、生產工作流程、評估覆蓋、低升級率、可量度改善、零關鍵安全事件同審計完整。

總結一下，呢個系統唔係再加幾個人工智能聊天框，而係將受治理、可審計、可評估、只喺沙盒演化嘅業務作業系統，落成後端控制平面同前端營運主控台 (ops console)。前端可以好清楚好靚，但政策、權限、基因同生產安全永遠喺後端。延伸可以睇 structure_hk、backend_hk、frontend_hk、planning/backend 同 planning/frontend 下面兩套 SDD，同三份 gap_analysis。
