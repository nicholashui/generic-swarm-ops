## 1. OWASP LLM Top 10（2025）控制措施

### 1.1 提示注入防禦

**核心原則：** 系統提示不是安全控制。在 LLM 外部確定性地強制執行安全。

**實施：**

- 工具權限代理強制執行工具存取控制
- 記憶體範圍在執行時強制執行
- 人工閘門在 LLM 之外確定性地啟動

### 1.2 資料洩露預防

```python
# 輸出過濾設定
OUTPUT_FILTERS = {
    "pii_detection": True,
    "credential_scanning": True,
    "scope_boundary_enforcement": True,
}
```

### 1.3 不安全的輸出處理

所有工具效果在執行前都經過驗證，`tool_effects` 記錄提供完整的審計軌跡。

## 2. OWASP 代理式 Top 10（2026）控制措施

### 2.1 過度代理（Excessive Agency）

**控制：** 有界狀態圖強制每個步驟只能存取宣告的工具。

### 2.2 不充分的存取控制

**控制：** 風險等級系統（Tier 0-5）限制代理的自主程度。

### 2.3 不安全的工具使用

**控制：** 工具權限代理提供窄範圍、暫時、限定的憑證。

---

## 3. 爆炸半徑控制

### 3.1 設計原則

間接提示注入無法完全預防，因此爆炸半徑控制是關鍵：

| 控制 | 機制 | 效果 |
|------|------|------|
| 工具範圍限制 | 每步驟宣告允許的工具 | 限制可能的操作 |
| 記憶體範圍隔離 | 命名空間強制執行 | 防止資料洩露 |
| 人工閘門 | 不可逆操作暫停 | 防止永久損害 |
| 回滾計劃 | 每個 DNA 必須宣告 | 確保可恢復性 |

### 3.2 最小權限實施

```yaml
# 每個步驟只宣告需要的工具
steps:
  - id: "send_email"
    agent: "business_orchestrator"
    tools: ["email"]  # 僅允許 email，不允許 crm、billing 等
```

## 4. 工具權限代理

### 4.1 憑證範圍

工具權限代理為每個任務提供：
- **窄範圍：** 僅限必要的操作
- **暫時性：** 執行後立即撤銷
- **限定性：** 僅對特定資源有效

### 4.2 設定

```python
TOOL_PERMISSION_CONFIG = {
    "credential_ttl_seconds": 300,
    "max_operations_per_credential": 5,
    "require_audit_on_use": True,
    "auto_revoke_on_step_complete": True,
}
```

## 5. 記憶體中毒防禦

### 5.1 來源追溯

每個記憶體寫入都攜帶來源追溯：

```json
{
  "content": "客戶偏好 email 通訊",
  "source": "workflow_run_abc123",
  "agent": "business_orchestrator",
  "confidence": 0.95,
  "human_verified": false
}
```

### 5.2 高影響寫入審查

對高影響記憶體寫入（影響組織級別知識）要求人工審查。

### 5.3 異常檢測

監控記憶體寫入模式，識別異常的批量寫入或內容偏差。

## 7. 事件回應

### 7.1 AI 事件回應手冊

| 事件類型 | 嚴重性 | 回應時間 | 操作 |
|---------|--------|---------|------|
| 提示注入嘗試 | 中 | 1 小時 | 記錄、分析、加強防禦 |
| 工具濫用 | 高 | 15 分鐘 | 撤銷憑證、暫停代理 |
| 記憶體中毒 | 高 | 30 分鐘 | 回滾記憶體、審查來源 |
| 資料洩露 | 嚴重 | 立即 | 隔離系統、通知利害關係人 |

### 7.2 事件後分析

每次事件後進行根本原因分析，更新防禦措施，並將教訓添加到組織記憶中。

## 9. 實際使用案例

### 使用案例 1：金融服務合規

**場景：** 銀行使用系統處理貸款申請，需要嚴格的資料安全。

**實施：** Tier 4 風險等級、所有決策需人工閘門、完整審計軌跡、PII 檢測和遮罩。

### 使用案例 2：醫療保健資料保護

**場景：** 醫院系統處理患者資料，需要 HIPAA 合規。

**實施：** 嚴格的記憶體範圍隔離、PHI 洩露測試、存取日誌保留 7 年。

## 10. 最佳實踐

1. **假設被突破：** 設計系統時假設提示注入會發生，專注於限制損害
2. **深度防禦：** 不要依賴單一控制措施
3. **確定性安全：** 在 LLM 外部強制執行安全規則
4. **持續驗證：** 定期執行安全掃描和紅隊測試
5. **證據驅動信任：** 代理的自主權必須通過證據贏得
6. **最小權限：** 永遠只授予完成任務所需的最少權限
7. **完整可觀察性：** 確保每個操作都被記錄和可追溯

## 11. 章節摘要

本章涵蓋了 Generic Swarm Ops 的安全強化與長期維護：

- **OWASP 控制措施：** LLM Top 10 和代理式 Top 10 的防禦實施
- **爆炸半徑控制：** 通過工具範圍、記憶體隔離和人工閘門限制損害
- **工具權限代理：** 窄範圍、暫時、限定的憑證管理
- **記憶體中毒防禦：** 來源追溯、人工審查和異常檢測
- **事件回應：** AI 特定的事件處理手冊
- **長期維護：** 定期安全審查、版本管理和合規性維護

核心原則是：系統提示不是安全控制。安全必須在 LLM 外部確定性地強制執行，爆炸半徑控制是抵禦間接提示注入的關鍵防線。
