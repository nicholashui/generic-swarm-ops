呢集我想同大家由頭到尾拆清楚呢套 Backend API Server 說明書。後端 as-built 有 Postgres 主控平面，入面用 runtime_state 呢種 JSONB 狀態、本機工具適配器 (tool adapters) 加可持久嘅工具副作用 (tool_effects)、流程智能磁碟產物、演化沙盒包括語料評估 (corpus)、金絲雀 (canary) 同回滾 (rollback)、自我改進包括反思 (reflect)、教訓庫 (lessons)、自動提案 (auto-propose)、迴圈 (loops) 同技能沙盒 (skill sandbox)、仲有 K1-lite、生產工作流程基因 (workflow DNA) 驗證，同埋 E1 操作者路徑。規格驅動開發包喺 planning 下面 backend 資料夾，有需求 (requirements)、設計 (design)、任務 (tasks)，由 BE-01 到 24，任務係第二點二版，每項都有交付程式路徑 (Deliverable code paths)。證據可以睇 status.md、backend 下面 README、gap_analysis_for_backend 落差報告一百分、任務對程式索引 TASK_TO_CODE_TRACEABILITY，同埋 e1 操作清單。前端契約喺 frontend.md 第三十三節，營運台只可以經應用程式介面版本一路由 (API /api/v1) 打嚟呢個後端。好，跟住我哋正式開始。

呢本書係將 backend_hk.md 講到人哋聽得明，適合業務負責人、營運主管、架構師、安全同治理人員、後端同全端工程師。佢主要答八條問題。第一，呢個後端到底係咩，點解前端唔可以直接掂代理 (agent) 同資料庫。第二，設計優先次序同原則點樣約束每個應用程式介面 (API)。第三，系統邊界同高層架構點樣將請求變成受治理執行。第四，認證、權限、使用者組織、代理同工具點組成安全底座。第五，工作流程基因 (workflow DNA) 點定義、執行、關卡、回滾同串流。第六，知識、記憶、評估、流程智能點服務執行層。第七，點解演化沙盒同自我改進永遠唔可以直接改生產。第八，倉庫入面規格分解同 as-built 程式點樣對返 backend_hk.md。內容順序係先定位後端角色，再目標技術棧邊界原則，再拆功能域同應用程式介面契約，最後補非功能、資料模型、完成定義，同埋第十四章實作對應，來源係計劃文件第二十四節。

而家講下第一章，後端係咩。最簡單講，後端應用程式介面伺服器 (Backend API Server) 係 structure.md 同 structure_hk.md 所講業務作業系統嘅受治理控制層。佢用安全嘅應用程式介面 (API) 對外暴露能力，等前端、管理主控台、命令列介面 (CLI)、自動化工具或者外部整合，可以安全咁同代理 (agents)、工作流程 (workflows)、知識同記憶 (knowledge and memory)、治理同審批 (governance and approvals)、評估 (evaluations)、審計日誌 (audit logs)、流程智能 (process intelligence)、演化沙盒 (evolution sandbox)，同埋自我改進迴圈 (self-improvement loops) 互動。

再嚟，前端絕對唔可以直接掂代理執行環境、資料庫 (databases)、工作流程引擎 (workflow engines)、大語言模型供應商 (LLM providers)、向量儲存 (vector stores)，同埋內部工具。所有存取都要經後端應用程式介面伺服器。呢度有段分層角色樣板，我哋唔讀原文，只講重點。前端係使用者體驗層，後端應用程式介面係受治理智能同控制層，代理係專門工作者，工作流程係結構化業務執行路徑，治理負責風險同審批，審計負責信任同可追溯。同架構文件嘅關係係咁：structure_hk.md 係整體架構願景權威來源，backend_hk.md 同呢本書講點樣變成可執行應用程式介面控制平面，frontend.md 講營運台只可以經應用程式介面操作，planning 下面 backend 就將計劃拆成 01 到 24 個子功能，每個都有需求設計同任務。

好啦，第二章，主要目標同技術棧。目標係整一個後端伺服器，封裝系統完整功能，用版本化應用程式介面 (versioned API) 對外提供。至少要支援安全驗證同角色為本存取控制 (RBAC)、代理編排同登錄、工作流程定義版本執行追蹤、人類審批關卡、記憶同知識檢索、審計日誌、評估同品質檢查、流程智能、演化沙盒由提出到評估到金絲雀到晉升或者回滾、自我改進包括反思教訓自動提案技能沙盒同迴圈執行器、生產基因安全檢查喺啟用或者標 production_ready 之前、背景非阻塞長任務、向前端串流更新，同埋可擴展整合。

接落嚟技術棧。呢度有段配置樣板，我哋唔逐行讀，只講用途。後端框架用 FastAPI，語言用 Python，資料庫用 PostgreSQL 做主控制平面，as-built 用 runtime_state 呢種 JSONB，向量搜尋可選 pgvector 等，佇列進階可以 Redis 加 Celery 或者 Temporal，但 as-built 係行程內執行引擎，認證用 JWT 或者 OAuth2 同 SSO 就緒，應用程式介面風格先 REST 再加伺服器推送事件串流 (SSE)，文件用 FastAPI 產生 OpenAPI。產品標竿本地唔強制 Docker。配好 Postgres 之後，JSON 檔只係備份同種子，唔係線上權威寫入。重點嚟啦，點解要應用程式介面優先 (API First)？因為前端同整合方只應該依賴公開契約，唔好依賴內部資料庫形狀或者代理私有介面，契約變更一定要版本化，例如應用程式介面版本一路由 (API /api/v1)。

而家嚟到第三章，系統邊界同高層架構。後端擁有同控制嘅包括應用程式介面存取同身份驗證、授權權限、代理同工作流程執行請求、治理檢查同審批關卡、審計事件、知識記憶存取、流程智能、評估、背景工作同整合適配器。後端唔係將請求盲目轉發去代理嘅薄代理，而係要強制執行權限、政策、工作流程規則、風險控制、可審計性同資料邊界。

跟住高層資料流。呢度有段架構流向樣板，我哋唔讀碼，只講流程。前端或者命令列或者整合先入 FastAPI 嘅應用程式介面版本一路由，經過身份驗證、授權、速率限制同請求識別碼 (request_id)，再入服務同運行時，然後分去治理審批、工作流程引擎按基因步驟行、工具適配器同工具副作用、知識記憶檢索、評估、只限沙盒演化、自我改進，最後審計同串流事件，底層持久就係 Postgres 嘅 runtime_state，可選再加向量、大語言模型或者物件儲存。同 structure_hk.md 六層銜接就係流程智能、知識、執行、演化、治理、安全全部由評估包裹，後端就係將呢啲層變成超文本傳輸協定 (HTTP)、執行引擎同持久狀態嘅落地點。

講完架構，第四章核心設計原則。七條工程原則分別係：應用程式介面優先，前端只依賴契約；預設安全，未明確公開端點一律要認證；治理優先，高風險先過政策同關卡；人在迴路 (human-in-the-loop)，不可逆或者高風險一定要人批；重要事項可審計，登入執行審批權限變更都寫審計；長任務非阻塞，異步或者工作者 (worker)，應用程式介面快啲交返執行識別碼 (run id)；前端簡化，智能控制安全信任強制喺伺服器。同架構優先次序一致，任何取捨都係安全性先，然後可審計性、正確性、效率，最後先到自主性。自主性唔係預設畀，要用評估證據同關卡逐步賺返嚟。

好，第五章安全底座。認證最低要有登入登出權杖刷新 (token refresh)、用密碼就有重設、機器客戶端用應用程式介面金鑰 (API key)、可選單一登入或者 OAuth 就緒。as-built 重點係密碼用 PBKDF2、已停用或者只係受邀狀態嘅使用者唔可以登入、敏感認證端點有速率限制 (rate limit)。授權方面用角色為本存取控制 (RBAC)，角色包括擁有者 (owner)、管理員 (admin)、經理 (manager)、操作者 (operator)、審核者 (reviewer)、檢視者 (viewer)、服務帳戶 (service account)。權限字串覆蓋代理、工作流程、執行、知識、記憶、治理、審批、審計、評估、設定、使用者、組織等。伺服器端強制，客戶端自己話有權唔算數。

再睇下使用者同組織。支援組織、使用者、角色、邀請、服務帳戶、應用程式介面金鑰。就算單租戶，資料模型都要有組織識別碼 (organization_id)。as-built 補充咗一啲路徑，我哋唔讀表，只講用途。可以列表同建立使用者，可以取得更新停用使用者，可以列表同建立邀請，有公開接受邀請端點，可以列表同更新組織。停用使用者會撤銷佢嘅有效權杖 (live tokens)。

接落嚟第六章，代理、工具同工作流程基因。每個代理要有識別碼、名稱、版本、允許工具、允許記憶範圍、風險等級、輸入輸出綱要 (schema)、狀態例如草稿、啟用、停用、封存。停用或者封存就唔可以再開新步驟，除非有明確覆寫政策。工具只可以經後端控制嘅權限用，高風險要治理規則或者審批。呢度有段中介交集樣板，我哋唔讀符號，只講規則。工具呼叫要同時過代理允許清單、基因步驟工具、角色權限，同埋人類關卡。as-built 執行用本機適配器，副作用寫入可持久工具副作用 (tool_effects)，失敗要封閉，唔好將錯誤當成功。外部即時客戶關係或者電郵軟件即服務 (live CRM or email SaaS) 屬非目標。工作流程定義係多步驟業務過程，包代理、條件、審批、記憶讀寫、工具、評估、輸出。狀態有草稿啟用停用封存。每次執行一定要釘死特定工作流程版本。啟用或者標 production_ready 之前要過生產基因驗證。

而家講下第七章，執行引擎。每次執行產生工作流程執行 (workflow_run)。狀態樣板我哋唔讀清單，只講有排隊、運行中、等審批、暫停、完成、失敗、取消、逾期、重試排隊等。要記輸入輸出錯誤、當前步驟、審批狀態、評估結果、成本用量。每步都有待處理、運行中、等審批、完成、失敗、略過、取消等狀態，仲有代理或者工具、輸入輸出、重試次數。

啟動流程規範順序係：先認證，再授權 workflows 執行權，載入工作流程版本，驗證輸入綱要，做治理預檢，建立排隊中嘅執行同審計，交返執行識別碼，引擎行步驟，前端輪詢或者伺服器推送事件串流 (SSE)。支援冪等鍵 (Idempotency-Key)，同一使用者同一把鎖唔會重複開執行。as-built 生命週期可以取消、暫停、由暫停恢復到排隊、逾期、重試，同埋串流。人類關卡就係高風險不可逆外部動作先建審批，執行入等審批，批準先繼續，拒絕就停門控動作。

講完執行，第八章治理審計知識記憶評估。治理層決定工作流程可唔可以跑、步驟可唔可以執行、工具可唔可以用、資料可唔可以讀、係咪要人批、輸出可唔可以放行。風險等級有低中高同關鍵 (critical)。政策動作有允許、拒絕、要求審批、要求評估、要求編修 (redaction)。審計方面，重要動作一定要寫，包括登入、工作流程變更、執行生命週期、審批、知識上傳、記憶敏感存取、權限變更、應用程式介面金鑰等。應用程式介面上審計唯讀，客戶端唔可以改歷史。知識庫支援上傳索引分塊嵌入搜尋存取控制清單同封存。as-built 分層檢索第零層預設，第一層多跳精簡版，結果要有來源出處 (provenance)，檢索內容當不可信資料。記憶系統多範圍，短期長期、使用者團隊部門組織工作流程代理等，強制存取控制同敏感度，代理唔可以讀寫未允許範圍。評估系統對重要輸出同工作流程做綱要、政策、安全、成本等檢查，必要失敗可以擋放行或者晉升，演化用語料評估打適應度。

好啦，第九章流程智能、演化沙盒同自我改進。流程智能提供指標、瓶頸、失敗、成本、審批延遲。as-built 聚合執行同事件資料，寫入業務流程智能資料夾產物，但唔可以直接改生產基因。演化沙盒有一條不可談判規則：演化管理員絕不能直接改生產，只可以提出變體、沙盒測試、同基線比較、請求批准、金絲雀、失敗自動回滾、通過先版本化晉升。呢度有段流程樣板，我哋只講呢個順序。應用程式介面強制只限沙盒 (sandbox_only)，禁止用 DGM 式改寫主機應用程式碼。自我改進方面，執行完可以自動或者手動反思，寫入教訓庫，再自動提案沙盒變體，仍然要行評估同金絲雀；技能寫入沙盒目錄先，要顯式晉升；迴圈基因執行器只做有界迭代。前端改進管線由反思到提案到評估到金絲雀，只可以打呢啲應用程式介面。生產基因安全用結構驗證器加業務驗證，查風險分級、關卡、回滾、來源欄位。驗證失敗就拒絕啟用或者 production_ready，可以記拒絕教訓，但唔好突變生產基因。

接落嚟第十章非功能同工程約束。安全上，受保護路由要身份驗證，資源要授權，要驗證請求本文同消毒上傳，密鑰唔好入程式庫、日誌同客戶端回應，敏感端點要速率限制，提示注入假設仍然可能，安全要喺大語言模型 (LLM) 之外用確定性控制，檢索用戶工具輸入輸出當資料唔當指令。可靠可觀測可維護方面，要優雅錯誤同安全錯誤信封，入面有請求識別碼，健康存活就緒探測，就緒要報資料庫狀態，可以強制要求 Postgres，仲要結構化日誌同指標，分層由路由到服務到領域到基礎設施。

而家嚟到第十一章應用程式介面契約同資料模型。公開前綴係應用程式介面版本一路由，成功回應風格一致有資料同元資料，錯誤有代碼訊息同請求識別碼，生產唔好回堆疊，OpenAPI 由 FastAPI 產生，前端用套件管理指令重新產生型別。主要端點族群包括認證、代理、工具、工作流程、執行、審批、知識、記憶、治理、評估、審計、流程、演化、改進、迴圈、設定、使用者、組織。核心實體概念有組織、使用者、代理、工作流程同版本、執行同步驟、審批請求、知識文件同分塊、記憶項、評估執行、審計日誌，同埋 as-built 嘅演化變體、教訓、工具副作用等集合。

第十二章資料夾同工程分層。呢度有段目錄樹樣板，我哋唔逐行讀，只講用途。後端應用目錄分入口、運行時門面、版本一路由、核心設定認證權限錯誤速率限制、領域邏輯、應用服務、基礎設施包括資料庫適配器檢索演化驗證器、綱要契約、同單元同端到端測試。任務對程式碼索引喺 planning 下面 backend 嘅 TASK_TO_CODE_TRACEABILITY。

第十三章完成定義同操作證明。首個生產就緒大概要認證、列表同啟動工作流程、輸入驗證、權限、審計、長任務、步驟執行、狀態同串流、審批關卡、知識搜尋存取控制、記憶控制、評估擋不安全輸出、OpenAPI、關鍵測試、部署文件、無硬編碼密鑰。產品標竿 mark 大約 100 再加六項，呢度有段清單樣板，我哋只講重點：有資料庫連線字串時 Postgres 做主持久，本機適配器寫工具副作用，演化強制只限沙盒路徑，自我改進反思教訓自動提案可用，啟用同 production_ready 要基因驗證，E1 端到端要由登入到執行到關卡到完成到改進。操作證明睇 e2e 測試同 e1 清單，單元同端到端以 status 同本地單元測試為準。

最後第十四章，實作對應，對返 backend_hk.md 第二十四節。文件關係方面，structure 同 structure_hk 係架構願景，planning 下面 structure 係架構規格驅動開發 01 到 17，backend 同 backend_hk 係後端計劃，planning 下面 backend 係後端子功能 01 到 24，backend 資料夾係 as-built FastAPI，frontend 係營運台，gap_analysis_for_backend 係任務對實作落差一百分。

能力關卡方面，甲階段基礎對應認證角色權限審計健康就緒種子同業務驗證，乙階段影子學習對應事件同流程應用程式介面、流程智能產物、知識攝取搜尋，丙階段受控副駕駛對應 Postgres 主存、基因執行、人類關卡、第零第一層檢索、適配器同基因檢查，丁階段演化沙盒對應語料評估、金絲雀回滾、自我改進迴圈同技能沙盒。

as-built 一覽係：控制平面 FastAPI 加 Postgres JSONB，JSON 做備份種子；工具用本機適配器加工具副作用；中介用允許清單交基因交角色權限交關卡；流程智能用服務加磁碟產物；檢索第零第一層精簡，完整商用圖稍後；演化只沙盒唔改主機程式；自我改進有反思教訓提案迴圈；基因安全用啟用前驗證器；操作證明有 E1；使用者組織有邀請停用同組織更新；執行生命週期有暫停恢復逾期。

後端子功能由 01 到 24，我哋用連貫方式講：憲章邊界原則、技術棧骨架、持久控制平面、應用程式介面契約錯誤信封、認證身份、授權角色、使用者組織租戶、代理登錄、工具適配器中介、工作流程定義版本、執行引擎、治理風險、人類關卡、審計、知識檢索、記憶、評估、流程智能、串流健康可觀測、演化沙盒應用程式介面、自我改進同迴圈、生產基因安全、安全加固同非功能、測試策略同操作路徑。每一項都有需求到設計到任務，任務有交付程式路徑。

明確非目標包括完整商用 LightRAG 同 Neo4j 生產網、外部即時客戶關係電郵帳單軟件即服務適配器、DGM 式主機自我改寫、常駐多工作者 Temporal 或者 Celery 作為硬性要求、每工具臨時 OAuth 中介、無限填滿業務資料夾每個葉節點。營運入口就係後端入口同運行時同基礎設施、資料庫連線字串對 Postgres、業務語料、延續檔、前端，同埋任務對碼索引。

總結一下，backend_hk.md 講嘅唔係再加幾個超文本傳輸協定路由，而係將受治理、可審計、可評估、只喺沙盒演化嘅商業作業系統，落成唯一可信嘅應用程式介面控制平面。前端可以好靚，但政策、權限、基因同生產安全永遠喺後端。延伸可以再睇 structure_hk 同架構說明書、backend_hk 同 backend.md、frontend 第三十三節、planning 下面 backend 說明、落差報告，同 status 同 backend README。
